🧩 Problem-Solving Templates

For each problem type: how to recognize it, the step-by-step process, the required formulas, an example setup, and common traps. Use these as decision-tree menus during the exam.

Template 1: "Formulate as LP" (Production / Allocation)

Recognize: "decide how much to produce / ship / allocate ... to maximize / minimize ..."

Process:
  1. Define decision variables — state what each means in words.
  2. Write objective: $\max / \min \sum c_j x_j$.
  3. Write resource constraints: $\sum_j a_{ij} x_j \le b_i$ (one per limited input).
  4. Write demand/balance constraints if applicable.
  5. Add non-negativity (and binary if relevant).
Traps: Forgetting non-negativity; missing a balance node; writing summations when problem says "no Σ"; mixing up units.

Template 2: Newsvendor

Recognize: "perishable / single-period / one-time stock decision" + uncertain demand.

Process:
  1. List parameters: $p$, $c$, $s$ (or $c_u$, $c_o$).
  2. Compute fractile $\alpha = c_u / (c_u + c_o)$.
  3. Identify distribution. Write CDF.
  4. Solve $F(Q^*) = \alpha$ via inversion.
  5. If asked for profit: integrate or use sample average.
Traps: Wrong quadratic root for triangular; forgetting salvage; misusing $z_\alpha$ sign.

Template 3: Choice / Assortment

Recognize: "customer segments with utilities" + "decide which products to offer."

Process:
  1. For each segment $k$: find max utility over $S \cup \{0\}$ (include no-purchase!).
  2. $P(i|S)$ = sum of $\lambda_k$ for segments choosing $i$.
  3. $R(S) = \sum_{i \in S} r_i \cdot P(i|S)$.
  4. For IP: $x_i$ (offer), $y_{k,i}$ (chosen), with 4 constraints + side conditions.
  5. For upper bound: customized (per-segment max revenue ≥ no-purchase) OR LP relaxation.
Traps: Forgetting no-purchase; confusing utility with revenue; forgetting $\lambda_k$ weights.

Template 4: LP Duality / Shadow Price

Recognize: Dual variables provided / "predict revenue change" / "buy more X?"

Process:
  1. For "predicted change from $\Delta b$": $\Delta R \approx \sum p_i^* \Delta b_i$.
  2. For "buy more": compare per-unit cost to shadow price.
  3. For "is primal optimal?": check primal feasibility, dual feasibility, then equal objectives.
  4. For complementary slackness: $p_i^* > 0 \Rightarrow$ constraint $i$ tight.
Traps: Forgetting weak duality requires BOTH feasible; ignoring sensitivity range.

Template 5: Network Flow LP

Recognize: Goods flowing through nodes with supply/demand/transshipment.

Process:
  1. Sketch the network: nodes, edges, costs, capacities.
  2. For each node, write flow conservation EXPLICITLY: inflow + supply = outflow + demand.
  3. If "no Σ notation": enumerate every variable in each constraint.
  4. For piecewise cost: tier var (capped) + excess var; update conservation.
Traps: Wrong direction of flow conservation; forgetting initial inventory; missing transshipment balance.

Template 6: Shortest / Reliability Path

Recognize: "find best path from A to L" with edge weights or reliabilities.

Process:
  1. If reliabilities: take $-\log r_{ij}$ to convert to additive minimization.
  2. Set source $b_s = +1$, sink $b_t = -1$, others $= 0$.
  3. Min $\sum c_{ij} x_{ij}$ s.t. flow conservation, $x_{ij} \in \{0,1\}$.
  4. For DP solve: $v_i = \min_j(c_{ij} + v_j)$ backward from sink.
  5. For top-K: iteratively forbid previous paths.
Traps: Log-reliabilities are negative — maximize = least negative.

Template 7: TSP

Recognize: "visit each location once and return to start."

Process:
  1. Build distance matrix (Manhattan or Euclidean per problem).
  2. Variables $x_{ij} \in \{0,1\}$, $i \ne j$.
  3. Min $\sum d_{ij} x_{ij}$ s.t. degree=1 in AND out.
  4. Add SEC: $\sum_{i \in S}\sum_{j \in S, j \ne i} x_{ij} \le |S| - 1$ (via lazy callback).
Traps: Forgetting both directions in SEC ($x_{ij}$ AND $x_{ji}$); wrong $|S|-1$ vs $|S|$.

Template 8: Location (MCLP / p-Median / p-Center)

Recognize: "open $p$ facilities" + (a) "within distance D" (MCLP) or (b) "minimize average" (p-median) or (c) "minimize max" (p-center).

Process:
  1. Define $x_j$ (open), $z_{ij}$ (assign), $y_j$ for MCLP (covered), $r$ for p-center.
  2. Write the model.
  3. Use the STRONG form: $z_{ij} \le x_j$ per pair.
  4. Add cardinality $\sum x_j = p$.
  5. Add side constraints from the universal patterns table.
Traps: Using weak form when strong is asked; mixing MCLP coverage with p-median assignment.

Template 9: Revenue Management (Bid-Price)

Recognize: "fare classes" / "capacity over time."

Process:
  1. Define $x_j$ = expected number of class $j$ to accept.
  2. Max $\sum r_j x_j$ s.t. $\sum x_j \le C$, $x_j \le E[D_j]$, $x \ge 0$.
  3. Solve LP; extract shadow prices $\pi_\ell$.
  4. Accept request iff $r_j \ge \pi_\ell$ (single-leg) or $\ge \sum_{\ell \in S_j} \pi_\ell$ (multi-leg).
  5. Re-solve LP at each arrival with updated $b$ and $T - t$.
Traps: Forgetting demand cap; using static allocation instead of dynamic; claiming policy > LP bound.

Template 10: Pricing IP

Recognize: "discrete prices to choose from" + linear demand parameters.

Process:
  1. Define $x_{i,t} \in \{0,1\}$: product $i$ at price level $t$.
  2. Add $\sum_t x_{i,t} = 1$ for each $i$.
  3. If cross-price terms: introduce $y_{i,j,t_1,t_2}$ with BOTH linking directions.
  4. Objective: baseline ($\alpha_i \cdot t \cdot x_{i,t}$) − own-price ($\beta_i \cdot t^2 \cdot x_{i,t}$) + cross ($\gamma_{i,j} \cdot t_1 t_2 \cdot y$).
Traps: Missing one direction of linking; using $t$ instead of $t^2$ in own-price; wrong sign on $\beta$.

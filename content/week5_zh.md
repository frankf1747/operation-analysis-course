# 第5周 — 品类选择优化 Assortment Optimization

## 0. 总体概览

你经营一家零售企业、餐厅或航空公司。你无法提供所有产品 —— 货架空间有限、菜单容量有限、运营约束有限。**问题:** 当客户存在替代行为时,应该提供哪个子集以最大化收入?

难点: 存在 $2^n$ 个可能的子集。但增加低利润产品会蚕食(CANNIBALIZE)高利润产品,因此"提供所有商品"很少是最优的。整数规划可以求解此问题。

## 1. 选择模型 — 基础

三个要素:
1. **客户是谁?** ($K$ 个客户细分,占比 $\lambda_k$)
2. **他们的偏好是什么?** (效用 $u_{k,i}$)
3. **他们如何决策?** (首选模型 / MNL模型 / 等等)

**选择概率:** $P(i, S)$ = 当提供 $S$ 时客户选择 $i$ 的概率。

**单个客户的期望收入:**
$$R(S) = \sum_{i \in S} r_i \cdot P(i, S)$$

**优化问题:** $\max_{S \subseteq [n]} R(S)$。

## 2. 最大效用模型(首选模型)

### 设定
- $K$ 个客户细分,权重为 $\lambda_k$ ($\sum \lambda_k = 1$)
- 每个细分 $k$ 对产品 $i$ 的效用为 $u_{k,i}$,对不购买选项的效用为 $u_{k,0}$
- **细分 $k$ 选择** $\arg\max_{i \in S \cup \{0\}} u_{k,i}$

### 选择概率
$$P(i | S) = \sum_{k: \arg\max u_{k,j} \text{ over } S \cup \{0\} = i} \lambda_k$$

通俗讲: 把那些将 $i$ 作为在 $S \cup \{0\}$ 中首选的细分的权重相加。

## 3. 葡萄酒精品店案例(来自练习题)

| 细分 | $\lambda$ | CabS(45) | Mer(20) | PN(17) | Syr(25) | Zin(18) | NP |
|---|---|---|---|---|---|---|---|
| 1 | 0.15 | 3.7 | 3.9 | 4.2 | 2.4 | 2.3 | 2.5 |
| 2 | 0.20 | 2.4 | 3.4 | 2.6 | 4.6 | 5.0 | 2.8 |
| 3 | 0.15 | 4.8 | 1.4 | 1.9 | 1.2 | 3.7 | 3.3 |
| 4 | 0.50 | 2.5 | 3.7 | 4.0 | 3.5 | 2.7 | 3.1 |

(括号中的数字为价格。)

### 计算 $S = \{3, 5\}$ = {Pinot Noir, Zinfandel}

对于每个细分,找出在 {3, 5, 0} 中的最大效用:
- 细分1: max(4.2, 2.3, 2.5) = 4.2 → PN(3)
- 细分2: max(2.6, 5.0, 2.8) = 5.0 → Zin(5)
- 细分3: max(1.9, 3.7, 3.3) = 3.7 → Zin(5)
- 细分4: max(4.0, 2.7, 3.1) = 4.0 → PN(3)

$P(3|S) = 0.15 + 0.50 = 0.65$。$P(5|S) = 0.20 + 0.15 = 0.35$。$P(0|S) = 0$。

$R(S) = 17 \times 0.65 + 18 \times 0.35 = 11.05 + 6.30 = 17.35$。

## 4. 为什么"提供所有商品"会失败

蚕食(Cannibalization,蚕食): 增加一个便宜产品会从昂贵产品那里抢走客户。

例子: 提供 {CS, PN}。某个细分以 $45 购买 CS。现在加入 Merlot —— 该细分可能会转而以 $20 购买 Merlot。收入下降!

Timbuk2案例: 提供全部100个产品 → 每位客户 $59.73。最优的39产品子集 → 每位客户 $76.80。当数量上限 ≤ 5 时甚至更好: $72.82。

## 5. 不求解 IP 的上界

### 上界1: 定制(按细分)上界
$$UB_1 = \sum_k \lambda_k \cdot \max_{i:\, u_{k,i} > u_{k,0}} r_i$$

**解读:** "如果我们可以为每个细分提供不同的品类组合。"过于乐观 —— IP 无法超过此上界。

**葡萄酒例子:**
- 细分1 在 NP 之上更喜欢 {CS, Mer, PN}(效用 > 2.5)。最佳收入 = $45 (CS)。
- 细分2 在 NP 之上更喜欢 {Syr, Zin}。最佳 = $25 (Syr)。
- 细分3 在 NP 之上更喜欢 {CS, Zin}。最佳 = $45 (CS)。
- 细分4 在 NP 之上更喜欢 {Mer, PN, Syr}。最佳 = $25 (Syr)。

$UB_1 = 0.15(45) + 0.20(25) + 0.15(45) + 0.50(25) = 6.75 + 5 + 6.75 + 12.5 = \$31$。

### 上界2: 线性松弛(LP Relaxation)
去掉0-1约束,允许 $x_i, y_{k,i} \in [0,1]$。求解 LP。LP 值 ≥ IP 最优值。

在 Timbuk2 中: LP = $81.69,IP = $76.80,差距为 6%。解接近最优。

## 6. 整数规划建模

### 决策变量
- $x_i \in \{0,1\}$: 是否提供产品 $i$
- $y_{k,i} \in \{0,1\}$: 细分 $k$ 是否选择 $i$(包含 $i=0$ 表示不购买)

### 目标函数
$$\max \sum_{k=1}^{K} \sum_{i=1}^{n} \lambda_k r_i y_{k,i}$$

### 约束条件
1. **每个细分恰好选择一个:**
$$\sum_{i=0}^{n} y_{k,i} = 1 \quad \forall k$$

2. **最大效用约束**(所选选项的效用 ≥ 任何可获得选项的效用):
$$\sum_{j=0}^{n} u_{k,j} y_{k,j} \ge u_{k,i} x_i + u_{k,0}(1-x_i) \quad \forall k, i$$

如果 $x_i = 1$: 所选效用必须不低于 $u_{k,i}$。
如果 $x_i = 0$: 所选效用必须不低于 $u_{k,0}$(不购买的回退选项)。

3. **始终优于不购买:**
$$\sum_{j=0}^{n} u_{k,j} y_{k,j} \ge u_{k,0} \quad \forall k$$

4. **只能选择已提供的产品:**
$$y_{k,i} \le x_i \quad \forall k, i \in [n]$$

5. **0-1约束:** $x_i, y_{k,i} \in \{0,1\}$。

## 7. 品类选择 Notebook —— 单元格逐行解析

**设定:** 330 位客户,100 个 Timbuk2 候选包(联合分析研究)。

```python
partworths = np.asarray(pd.read_csv("partworths_small_v2.csv"))  # (330, 10)
grand_product_matrix = pd.read_csv(...)  # (10, 100)
utilities_mat = np.dot(partworths, grand_product_matrix)  # (330, 100)
```

**不购买的效用** = 来自 3 个竞争对手产品的最大效用:
```python
competitive_utilities = np.dot(partworths, competitive_products.T)  # (330, 3)
nopurchase_utilities = competitive_utilities.max(axis=1)
utilities_mat = np.concatenate((utilities_mat, nopurchase_utilities[:,None]), axis=1)  # (330, 101)
```

**每个产品的边际利润** = 价格 − 各属性成本。

**暴力枚举评估器:**
```python
def expected_profit(S):
    S2 = np.append(S, nProducts)  # 加入不购买选项
    prob = {i: 0.0 for i in S2}
    for k in range(330):
        best = np.argmax(utilities_mat[k, S2])
        prob[S2[best]] += 1/330
    return sum(prob[i] * marginal_profit[i] for i in S)
```

**Gurobi IP** 包含全部 4 个约束。求解 → 最优 39 种产品,每位客户 $76.80。

**当数量上限 ≤ 5 时:** 加入约束 `m.addConstr(sum(x[i] for i in range(nProducts)) <= 5)`。最优 {0, 18, 33, 52, 73},每位客户 $72.82。

## 8. 附加约束清单

| 需求 | 约束 |
|---|---|
| 提供 $m$–$M$ 个产品 | $m \le \sum_i x_i \le M$ |
| $i$ 被 ≥ 20% 客户选择(绝对值) | $\sum_k \lambda_k y_{k,i} \ge 0.20$ |
| 若提供 $i$,则 $i$ 被 ≥ 20% 选择 | $\sum_k \lambda_k y_{k,i} \ge 0.20 x_i$ |
| 若提供 $i$,则必须至少提供 {a,b,c} 中的 1 个 | $x_i \le x_a + x_b + x_c$ |
| 若同时提供 $i$ 和 $j$ → 也必须提供 $m, n$ | $x_i + x_j - 1 \le x_m$,$x_i + x_j - 1 \le x_n$ |
| 互斥 | $x_i + x_j \le 1$ |

## 9. MNL 模型(如有涉及)

$$P(i|S) = \frac{e^{u_i}}{e^{u_0} + \sum_{j \in S} e^{u_j}}$$

首选模型的概率化替代方案。更现实但非凹 → 更难优化。

## 10. 分步方法

### (a) 选择概率的计算
1. 对每个细分,找出 $j \in S \cup \{0\}$ 中 $u_{k,j}$ 的最大值。
2. 确定该最大值对应的选项。
3. 对选择同一选项的所有细分,把 $\lambda_k$ 累加。

### (b) 期望收入
$R(S) = \sum_{i \in S} r_i P(i|S)$。

### (c) 定制上界
对每个细分,找出满足 $u_{k,i} > u_{k,0}$ 中收入最高的产品。按 $\lambda_k$ 加权,求和。

### (d) 建立 IP
变量 ($x_i, y_{k,i}$) + 4 个核心约束 + 附加约束。

### (e) 线性松弛
将 `vtype=GRB.BINARY` 改为 `vtype=GRB.CONTINUOUS`,边界设为 [0,1]。

## 11. 常见错误 ⚠️

1. 在 argmax 中忘记了不购买选项。
2. 把效用 $u_{k,i}$ 与收入 $r_i$ 混淆。
3. 没有按 $\lambda_k$ 加权。
4. 忘记"若提供"的限定条件(条件 vs 无条件选择概率)。
5. 逻辑约束错误: "如果 A 且 B,则 C" → $x_A + x_B - 1 \le x_C$,而非 $x_A + x_B \le x_C$。
6. 把 MNL 当作首选模型来处理。
7. 误以为"提供所有产品"是最优的。
8. 假设最高利润的产品总会出现在最优组合中。

## 14. 速查卡

| 概念 | 公式 |
|---|---|
| 选择概率(首选模型) | $P(i|S) = \sum_{k:\arg\max = i} \lambda_k$ |
| 期望收入 | $R(S) = \sum r_i P(i|S)$ |
| 定制上界 | $\sum_k \lambda_k \max_{i: u_{k,i}>u_{k,0}} r_i$ |
| MNL 选择 | $P(i|S) = e^{u_i}/(e^{u_0}+\sum_{j\in S} e^{u_j})$ |

**IP 模板:**
- max $\sum_k \sum_i \lambda_k r_i y_{k,i}$
- $\sum_i y_{k,i} = 1 \forall k$
- $y_{k,i} \le x_i \forall k, i$
- 最大效用约束
- 优于 NP 的约束

**关键直觉:**
- 替代 → 蚕食 → "全部提供"并非最优。
- 异质细分 → 不存在单一最佳产品。
- 不购买始终是一个竞争对手。
- LP 松弛提供上界;差距小 = 接近最优。

## 🎙️ 课堂录音洞察 —— 幻灯片之外的内容

来自 Misic 教授 4/29 的课堂。

### 核心洞察(反复强调)

**不购买选项是必须的。** 与幻灯片侧重不同,录音中反复强调: 客户始终拥有离开的选项。如果你提供 Timbuk2 包但客户都不喜欢 → 他们就会从竞争对手处购买(如 North Face)。**外部选项**的效用 = 竞争对手产品的最大效用。

### 为什么直觉策略会失败(核心张力)

**提供所有商品 ≠ 最优:** 提供全部 100 个 Timbuk2 产品 → 期望利润 = $59.73。最优子集 → $76.80。为什么?

**具体例子(手机):** 在 {3} 基础上加入产品 4,使收入从 $150 增加到 $175。原因:
- 如果只有产品 3 可用 → 类型 1 客户买它 ($200,λ=0.5 → $100)。
- 加入产品 4 之后 → 类型 3 客户买产品 4(利润增加大于轻微蚕食的损失)。
- **企业利用"排除"来推动有利的替代:** 移除高效用/低利润的产品,从而让客户向高利润产品"降级"购买。

**只选最高利润产品会失败:** 利润前10的产品 → 仅 $27.26(对比全部产品的 $59.73)。为什么?只有极少数客户对高利润产品有高效用。大多数客户会流失到竞争对手。

### 细分异质性才是难点

**Timbuk2 调研(330 位 MIT MBA 学生,16 次配对比较):**
- 客户 9: 非常喜欢 PDA 收纳袋(+11.79 效用),非常不喜欢魔术贴翻盖(-4.45)。
- 客户 19: 完全相反。

> *"如何设计一款双方都喜欢的产品?答案是: 做不到。你必须提供一个精心挑选的产品组合,并接受某些客户的妥协。"*

### 蚕食逻辑(重复3次)

**增加一个产品:**
- 一些非购买者转为购买(好)。
- 一些原有的购买者切换到新产品(不好)。
- 当转化收益 > 蚕食损失时,净利润上升。

**等式:** 如果新产品 X 转化了 20 名非购买者,但从 Y 抢走了 5 名:
- 当 margin(X) > (5/20) × margin(Y) 时净增长为正。

### Part-Worth 效用与美元解读

**来自联合分析的 330 × 10 part-worth 矩阵。**
- 每个元素是一个系数(类似回归斜率)。
- 价格 part-worth 始终为负。
- 归一化: 把价格的 part-worth 设为 −1,则其他系数 = 以美元表示的支付意愿(WTP)。

**例子:** 如果归一化后 part-worth(大号尺寸) = 21,则客户愿意为大号包多支付 $21。

### 首选模型 vs MNL 的对比

**首选模型(今天的重点):**
- 对每种类型来说是确定的。如果 utility(3) > utility(2),该类型总会选 3。
- 异质性 = 多种客户类型(330 个客户 = 330 种类型,每种 λ_k = 1/330)。
- 即使效用非常接近 → 仍然是确定性的(总有一个胜出)。

**MNL(提到但作为对比):**
- 即便效用固定也是随机的(softmax)。
- 效用接近 → 概率上分散选择。

**考试:** 若被要求批判首选模型,可以提到"确定性 —— 无法刻画效用接近时的不确定性"。

### 学生问答

**问: 如果客户全都不同,价格敏感性还重要吗?**
答: 重要。价格 part-worth 在不同客户间不同。按价格归一化 → 揭示真实的 WTP 异质性。

**问: 我们能否惩罚竞争对手的市场份额?**
答: 很难 —— 一旦客户不从我们这买,他们就会选择效用最高的竞争对手;我们无法控制。但可以增加一个次要目标。

**问: 如果效用非常接近怎么办?**
答: 首选模型是确定性的 —— 总有一个胜出。这是相对于 MNL 的局限。

### 最优解的性质 —— 不存在有利的交换

对于手机的例子,{3, 4} 是最优的,因为:
- 类型 1: 在 NP 之上可买 {1, 2, 3} → 选 3(可行选项中利润最高)。
- 类型 2: 在 NP 之上可买 {1, 4} → 选 4(可行选项中利润最高)。
- 类型 3: 可买 {4} → 选 4。

**没有任何交换能提高总利润。** 这就是最优性检验。

**考试技巧:** 求解(或猜测)出某品类组合后,验证: 对每种类型来说,能否通过移除其他产品来选择更高利润的产品?如果不能,则(局部)最优。

### Timbuk2 案例的运营设定

- 调研了 330 位高管 MBA 学生
- 16 次成对比较的包
- 价格(70-100,以 $5 为递增 = 7 个档位)+ 9 个二元特征(尺寸、颜色、Logo、提手、PDA 收纳袋、手机袋、网布/魔术贴、防护底)
- 完整设计空间: 7 × 2^9 ≈ 3,600 个产品(无法处理)
- Notebook 中限制为 100 个候选
- 三个竞争对手包(低端 $70、中端 $85、高端 $100)定义每种类型的不购买效用

### 口头反复强调的警告

1. **不要忘了不购买选项。** 考试中最常见的错误。
2. **替代 ≠ 流失。** 排除 A: 一些买家转向 B(好),另一些流失到竞争对手(不好)。
3. **蚕食是双刃剑。** 加入高利润产品可能蚕食低利润产品;期望利润可能下降。
4. **利润 ≠ 效用。** 客户最大化效用;你最大化利润。这种不一致就是整个博弈的关键。

### 策略: 如何搭建考试问题

1. 加载效用(客户 × 产品矩阵)。
2. 为每位客户定义不购买效用(给定或推断)。
3. 对每个候选品类组合 $S$:
   a. 对每种客户类型: 选择 = argmax(在 $S \cup \{0\}$ 中的效用)。
   b. 汇总选择概率: $P(i) = (\text{选择 } i \text{ 的类型数}) / N$。
   c. 期望利润 $= \sum_i \text{margin}(i) \times P(i)$。
4. 在所有品类组合中搜索(规模小: 暴力枚举;规模大: IP)。

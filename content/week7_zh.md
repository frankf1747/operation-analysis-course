# 第七周 — 网络流问题

## 0. 总体概览

包裹运输、电网、血液供应链 —— 这些都是由节点通过边连接而成的网络。**网络流问题** 优化什么、多少流量、流向何处。**神奇之处**:它们的 LP 松弛总是有整数最优解(**全幺模性**)。所以我们以 LP 求解,却能免费获得整数答案。

本周统一了:**最短路径** (最短路径)、**最小成本网络流** (最小成本网络流) 和 **运输问题**,以及带可靠性和附加约束的变体。

**为什么本周很特殊:** 这是整学期最"美"的一周——一类**结构特殊**的 IP,可以用 LP 求解。**全幺模性(TU)** 是一个数学定理:它保证只要你的约束矩阵满足某些性质,LP 的极点解(顶点)自然就是整数。这意味着:
- 不需要 Gurobi 的 IP 算法。
- 不需要分支定界。
- 几千个节点的网络,几毫秒就能解。

但这种神奇**很脆弱**——加一条"非流守恒"的约束(比如"A 和 B 不能同时用"),TU 立刻失效,必须用 IP。后面会演示。

## 1. 图论基础

- **节点**(顶点, 节点):城市、仓库、交叉点
- **边**(弧, 边):有向连接(A → B 表示流量可以从 A 流向 B,但反向除非有单独的边)
- **源**(源):具有供应的节点(产生流量)
- **汇**(汇):具有需求的节点(消耗流量)
- **转运点**(转运点):无供应或需求;流量经过
- **容量** $u_{ij}$:边上的最大流量
- **成本** $c_{ij}$:每单位运输成本

**画图记忆:** 把节点画成圈、边画成箭头,把每个节点上方写上"供应/需求/转运"(用 +X, -X, 0 标注),每条边写上"成本 / 容量"。**清晰的图 = 清晰的 LP**。考试拿到题先画图,再开始写公式。

## 2. 最小成本网络流 (MCNF)

### 设置
- 有向图 $(N, E)$
- 边 $(i,j)$:成本 $c_{ij}$,容量 $u_{ij}$
- 节点 $i$:供应 $b_i$ ($> 0$ = 供应, $< 0$ = 需求, $= 0$ = 转运点)
- **平衡条件: $\sum_i b_i = 0$**

**为什么必须平衡?** 因为"流入量 = 流出量"——所有供应必须等于所有需求。如果 $\sum b_i > 0$,意味着供应过剩,LP 不可行(找不到把所有货送掉的方案)。$\sum b_i < 0$ 意味着需求大于供应,也不可行。

**实务对策**:如果实际情况不平衡,引入"虚拟"节点或"短缺"变量来处理多余/不足。后面 §7 会演示。

### 决策变量
$x_{ij}$ = 边 $(i,j)$ 上的流量。

### 流守恒 流守恒 (关键约束)
在每个节点 $i$:
$$\sum_{j: (j,i) \in E} x_{ji} + b_i = \sum_{j: (i,j) \in E} x_{ij}$$
**流入 + 供应 = 流出 + 需求**

等价地:**净流出 = 供应**。

**这是本周最重要的方程**。逐项解读:
- LHS:流入节点 $i$ 的所有流量 + 节点自己的供应。
- RHS:从节点 $i$ 流出的所有流量(隐含包括"需求"——如果 $i$ 是汇,流出有部分被"消费"掉)。

写成"净流出 = 供应"的版本:**节点 $i$ 流出的减去流入的 = $b_i$**。这是更对称的写法,源(+)、汇(-)、转运(0)用同一个公式。

**很多学生在这里出错**——记住:**等号两边都要对**。多写一项或漏一项,整个 LP 都会失败。

### 完整 LP
$$\min \sum_{(i,j) \in E} c_{ij} x_{ij}$$
$$\text{s.t. 每个节点的流守恒}$$
$$0 \le x_{ij} \le u_{ij}$$

**就这么简单**——三行写完。复杂的全在如何"建图"——把文字题翻译成节点 + 边 + 供应/需求。一旦图建对了,LP 是机械的。

## 3. 全幺模性 完全幺模性

**定理:** 如果约束矩阵是 TU 且所有右端项为整数,则每个基本可行解都是整数。

**含义:** 纯网络流 LP → 最优 $x_{ij}$ 自动为整数。无需分支定界。

**为什么这是"奇迹"?** 因为整数规划通常是 NP-hard——理论上指数级时间。但 TU 把它"降级"为 LP,多项式时间。原因:网络流约束矩阵的特殊结构(每列恰好两个非零项:一个 +1 一个 -1)使得每个子矩阵行列式必为 ±1 或 0。

**这不是技术细节——是网络流问题最深刻的事实**。运输问题、最短路径、最大流、二部匹配,所有这些经典优化问题都有 TU 性质,所以都能用 LP 解。

**失效条件:** 添加非流守恒约束(例如, $x_{B,F} + x_{J,K} \le 1$)。此时需要 IP。

**为什么失效?** 因为新约束破坏了"每列两个非零项"的结构。加完之后,LP 可能给出分数解(0.5)——你不能"用半条边"。

## 4. 运输问题(特殊 MCNF)

- 二部图:源 → 汇 直接相连(无转运点)
- $m$ 个供应节点, $n$ 个需求节点
- $m \times n$ 条边
- 特殊情况可由专门算法求解(西北角法、Vogel 法)

**何时识别为"运输问题"?** 题目特征:**没有中间节点**。比如"3 个工厂直接发货到 4 个客户"。如果中间有仓库、配送中心、转运站,就是一般 MCNF。

**专门算法的存在意义**:运输问题结构特别简单,可以用比 LP 更快的专门算法解。考试一般不要求你手动跑这些算法,但要能识别问题类型。

## 5. 最短路径作为 MCNF

求 $s \to t$ 的最小成本路径:
- $b_s = +1$, $b_t = -1$,其他 $= 0$
- $u_{ij} = 1$(或 $\infty$ —— LP 会选择二进制)
- 最小化 $\sum c_{ij} x_{ij}$

$x_{ij} = 1$ 的边构成最短路径。

**这是 TU 的最优美的应用**:把"找最短路径"翻译成"送 1 单位货从 $s$ 到 $t$,最小化运费"。由于 TU,$x$ 自动是 0/1——构成路径。

**直觉**:**1 单位货只能走一条路**——它进入起点,沿途经过若干边,最后到达终点。每条用过的边 $x_{ij} = 1$,没用的 = 0。流守恒保证"中间节点不能堆积"——进入了就要出去。

## 6. 最短路径笔记本 —— 单元格演练

11 个节点,17 条边。源 A,汇 K。

```python
edge_data = [("A","B",3), ("A","C",5), ...]
m = Model()
x = m.addVars(edges, lb=0, ub=1)
# A: must leave
m.addConstr(sum(x["A",j] for j in outgoing(A)) == 1)
# K: must arrive
m.addConstr(sum(x[i,"K"] for i in incoming(K)) == 1)
# Others: balance
for i in middle_nodes:
    m.addConstr(sum(x[i,j] for j) == sum(x[j,i] for j))
m.setObjective(sum(time[i,j]*x[i,j] for (i,j) in edges), GRB.MINIMIZE)
m.optimize()
# Optimal: A → B → F → J → K, cost 9
```

**关键技巧**:`x = m.addVars(edges, lb=0, ub=1)` 没有声明 `vtype=GRB.BINARY`。变量是**连续**的 [0, 1] 之间。但由于 TU,最优解恰好是整数 0/1——免费拿到整数解。

**对比之前的 IP**:之前几周的 IP 必须 `vtype=GRB.BINARY`,LP 松弛是"作弊"工具(给个下界)。这里 LP 就是最终答案——不需要分支定界。

**添加非网络约束:** $x_{B,F} + x_{J,K} \le 1$。LP 现在给出分数解 $x_{B,F} = 0.5$(TU 被破坏)。

**这就是 TU 失效的演示**:加一条"非网络"约束,LP 立刻退化——给你一个分数解 $x_{B,F} = x_{J,K} = 0.5$,满足新约束(0.5 + 0.5 ≤ 1),但**不构成路径**——0.5 单位货物没意义。这时必须 `vtype=GRB.BINARY` 让 Gurobi 走 IP。

## 7. MCNF 笔记本 —— 演练

2 个工厂、5 个仓库、2 个零售商,共 14 个节点。加载 CSV,为每个节点构建流守恒。求解 → 成本 \$8,860。

**扩展生产决策:**
- $y_n$ = 工厂 $n$ 的产量(变量)
- $z_n$ = 零售商 $n$ 处的短缺(允许的未满足需求)
- 零售需求增加了 50%(不平衡)
- 最优解: $y_{P1} = 320, y_{P2} = 240, z_{R1} = 100$。成本 \$18,360。

**这个扩展演示了如何处理不平衡**:当需求超出供应,我们**不放弃问题**,而是允许"短缺" $z$ —— 给它一个高惩罚成本(比如失去销售的利润损失)。LP 自动权衡:多生产 vs 多短缺,哪个总成本低。

**容量约束上的影子价格** 揭示了瓶颈。边 (P1, A1): $\pi = -10$。将容量扩大 50 可节省约 \$500。使用二进制 $w_e$ 来选择扩展哪条边。

**影子价格的意义**:$\pi = -10$(最小化问题里影子价格 ≤ 0)意味着"每多 1 单位容量可降低 10 的成本"。**这就是第一周影子价格的直接应用**——网络流里它告诉你"哪条边是瓶颈,扩张该边收益多大"。

**实务用法**:把所有边的 $\pi$ 排序,最负的就是最紧迫扩张的。$w_e$ 二元变量加"扩 / 不扩"约束 → 进一步问"在预算下选哪几条边扩"。

## 8. 生产 + 运输 练习题(网络流-I)—— 核心例题

### 问题:Psyche 鞋业公司
- **生产工厂:** Dallas (\$10/单位), Albuquerque (\$12/单位)
- **配送中心:** Phoenix(库存 100), Las Vegas(库存 0)
- **零售店:** LA(需求 260), SF(需求 450)
- **第三方订单:** 200 单位直接从 Dallas 履行
- **LA/SF 以外的在线需求:** 510 单位,从 Las Vegas 履行

**运输成本(\$/单位):**
- D→LV: 5, D→P: 7, A→LV: 4, A→P: 9
- P→LA: 3, LV→LA: 3, LV→SF: 5

### 完整 LP(无 Σ 表示 —— 考试格式)

**变量:** $y_D, y_A$(生产); $x_{ij}$(每条边上的流量)。

**目标函数:**
$$\min 10y_D + 12y_A + 5x_{D,LV} + 7x_{D,P} + 4x_{A,LV} + 9x_{A,P} + 3x_{P,LA} + 3x_{LV,LA} + 5x_{LV,SF}$$

**约束条件:**
- Dallas: $y_D = 200 + x_{D,LV} + x_{D,P}$
- Albuquerque: $y_A = x_{A,LV} + x_{A,P}$
- Las Vegas: $x_{D,LV} + x_{A,LV} = 510 + x_{LV,LA} + x_{LV,SF}$
- Phoenix: $x_{D,P} + x_{A,P} + 100 = x_{P,LA}$
- LA: $x_{P,LA} + x_{LV,LA} = 260$
- SF: $x_{LV,SF} = 450$
- 所有 $\ge 0$

**逐条流守恒解读:**
- **Dallas**:$y_D$ 是 Dallas 生产的总量。这些货 200 单位送给第三方(题外),剩下流出到 LV 或 P。"流出 = 供应"形式。
- **Albuquerque**:同理。注意 A 没有第三方需求,所以 $y_A$ 全部送出。
- **Las Vegas**:进来的(从 D 和 A)= 在线需求 510 + 流出到 LA 和 SF。注意 LV **初始库存 0**。
- **Phoenix**:进来的 + **初始库存 100** = 流出到 LA。
- **LA, SF**:流入 = 需求。终点节点,没有出去的边。

**"数字来源":**
- Dallas 的 200 = 第三方需求
- LV 的 510 = 所有 LA/SF 以外的在线订单总和 (150+200+90+70)
- Phoenix 的 100 = 初始库存
- LA 的 260 = 零售 (200) + 在线 (60)
- SF 的 450 = 零售 (100) + 在线 (350)

**为什么考试要求"无 Σ"?** 因为 Σ 容易写错(指标范围、嵌套),展开后**每条变量都明确出现**,改错容易。考试想测试的是"你能不能完整列举每个节点的流"——展开形式逼你这么做。

## 9. 分段线性成本

"前 100 单位 P→LA 价格 \$3,超过部分 \$6。"

**修改方式:**
1. 添加新变量 $\tilde x_{P,LA}$(价格 \$6 的超额流量)。
2. 限制 $x_{P,LA} \le 100$。
3. 修改 Phoenix: $x_{D,P} + x_{A,P} + 100 = x_{P,LA} + \tilde x_{P,LA}$。
4. 修改 LA: $x_{P,LA} + \tilde x_{P,LA} + x_{LV,LA} = 260$。
5. 在目标函数中加入 $+6 \tilde x_{P,LA}$。

**为何有效:** 最小化成本时,LP 会先把便宜的 $x_{P,LA}$ 填满到 100 的上限,然后才使用 $\tilde x$。

**这是网络流的一个标准"trick"**:把一条边按价格档位拆成多条平行边,每条限制上限。LP 由于最小化,**自动按"便宜在前"使用**——不需要额外的逻辑约束。

**何时这一招失效?** 如果价格阶梯**先贵后便宜**(批量折扣)——比如"前 100 单位 \$6,超过 \$3"——LP 不会"自动选便宜"。这种凹成本结构需要二元变量,破坏 TU。这就是为什么本周的分段都是"先便宜后贵"。

## 10. 最大可靠性路径 —— 练习题 网络流-II

### 问题
管道 A → L。每条边有可靠性 $r_{ij} \in (0, 1)$。路径可靠性 = 各边可靠性的乘积。

### 关键见解:对数变换
$$\max \prod r_{ij} = \max \log \prod r_{ij} = \max \sum \log r_{ij}$$

由于当 $r_{ij} < 1$ 时 $\log r_{ij} < 0$,总和为负。最大化总和 = 最不负的路径。

**等价于:** $\min \sum (-\log r_{ij})$,其中 $-\log r_{ij} > 0$ → 最短路径。

**这是本周最优雅的 trick**:**乘积**问题用对数变成**求和**问题,再变成**最短路径**问题。一招把非线性变线性,把陌生题变熟悉题。

**为什么对数能做这种事?** 因为 $\log$ 把乘法变加法(基本性质)。如果你想最大化 $r_1 \cdot r_2 \cdot r_3$,等价于最大化 $\log r_1 + \log r_2 + \log r_3$(因为 $\log$ 是单调的)。再加个负号,变成最小化——就是经典最短路径。

### 公式化
带有边成本 $c_{ij} = -\log r_{ij}$ 的最短路径 IP。

### 动态规划(反向递归)
$v_i$ = 从 $i$ 到 $L$ 的最大对数可靠性。
$v_L = 0$。
$v_i = \max_{j \in O(i)} \{\log r_{ij} + v_j\}$。

**练习题结果:** $v_A = -2.3$。最大可靠性 = $e^{-2.3} = 0.100$。路径: A → C → D → H → J → L。

**DP 的工作方式(反向递归):**
1. 从终点 L 开始,$v_L = 0$($L$ 到 $L$,可靠性 1,对数 0)。
2. 对每个直接到 L 的节点 $i$,$v_i = \log r_{iL} + 0$。
3. 往回推,每个节点取"在所有可能下一步里使 $\log r + v$ 最大的那个"。
4. 直到回到起点 $A$。

DP 的特点是**多项式时间**——不需要枚举所有路径($2^n$ 量级)。

### 路径上的附加约束
- "如果 C→E 则不能 G→J": $x_{C,E} + x_{G,J} \le 1$
- "如果 A→C 且 C→E 则 J→L": $x_{A,C} + x_{C,E} - 1 \le x_{J,L}$
- "如果经过 E(使用 E→G), ≥ 4 条边": $4 x_{E,G} \le \sum_{(i,j)} x_{ij}$

**注意第三条**:"$\ge 4$ 条边" 是在 $E$→$G$ 被使用时($x_{E,G} = 1$)。LHS 用 $4 x_{E,G}$ 来"激活"约束——如果 $x_{E,G} = 0$,LHS = 0,约束自动满足;$x_{E,G} = 1$,LHS = 4,强制 $\sum x \ge 4$。

### 前K条最短路径(迭代)
1. 求解,获得路径 1 的边集 $E^{*,1}$,长度 $L_1$。
2. 添加: $\sum_{(i,j) \in E^{*,1}} x_{ij} \le L_1 - 1$。
3. 重新求解 → 路径 2。
4. 重复 $k$ 次。

**这一招在做什么:** "禁用上一条路径"——加约束"不能完整复用上次的所有边"($\sum x \le L_1 - 1$ 意味着至少有一条边和上次不一样)。重新解 LP,得到第二最短。

**实务意义**:在路径规划里,你可能不只要最短路径,还要"前 3 个备选路径"——以防最短的中途不能走。这就是这个算法的用处。

## 11. 分步方法

### (a) 将文字题转化为网络
1. 每个实体 = 节点。
2. 每条运输路线 = 有向边。
3. 供应 (+), 需求 (−), 转运点 (0)。
4. 每条边的成本和容量。

### (b) 明确写出流守恒
对每个节点 $i$: $\sum_j x_{ji} + b_i = \sum_j x_{ij}$。列出所有变量。

### (c) 分段成本
将边拆分为各层级变量;对每个限制上限;在守恒中求和。

### (d) 最短路径
$b_s = 1, b_t = -1$,其他为 0; 最小化 $\sum c_{ij} x_{ij}$。

### (e) 最大可靠性
与最短路径相同,但 $c_{ij} = -\log r_{ij}$。

### (f) 路径附加约束
关于 $x_{ij}$ 的线性约束(破坏 TU;可能需要 IP)。

**6 个子流程,覆盖本周所有题型**:把文字题归类到 a-f,然后机械应用。

## 12. 常见错误 ⚠️

1. **流守恒符号错误。** 净流出 = 供应。
   - LHS 写流入还是流出?各位老师约定不一样——务必看清题目里的形式。

2. **忘记初始库存。** Phoenix 手头有 100;包括它。
   - 如果一个节点有初始库存,把它当"额外供应"加进去。

3. **混淆平衡与非平衡。** $\sum b_i$ 必须等于 0(或具有短缺/过剩变量)。
   - 不平衡的题目要先决定:加供应/需求虚拟节点,还是用 $z$(短缺)/$w$(过剩)变量。

4. **边的方向错误。** A → B 是有向的。
   - 如果题目说"双向运输",得加两条边 (A→B) 和 (B→A)。

5. **忘记非负性。**

6. **对数可靠性符号混淆。** 对数是负的;最小化负值。
   - 最大化 $\sum \log r$ = 最小化 $\sum (-\log r)$,后者系数为正。

7. **没有二进制就添加非网络约束。** 破坏 TU。

8. **不理解为何分段有效。** LP 贪婪地使用更便宜的选项。

## 13. 考试相关性

考试幻灯片列出了 2 种题型:
1. **供应链 LP:** 文字题 → 流守恒 → 带有明确数字的完整 LP(无 Σ)。
2. **路径问题:** 最短/最长/可靠性,带附加约束、前K条。

练习题: 网络流-I(生产 + 运输)、网络流-II(最大可靠性 + 附加约束 + 前K条)。

**应对策略**:
- 类型 1:先画图(节点 + 边 + 供应/需求),然后对每个节点写流守恒,展开所有变量。
- 类型 2:识别"最短" / "可靠性" / "附加约束" / "前 K",对应模板套。

## 14. 小练习

**Q1.** 节点 {A,B,C,D}。边: A→B(2), A→C(5), B→D(3), C→D(1)。最短 A→D 路径?

$b_A = 1, b_D = -1, b_B = b_C = 0$。最优解: $x_{AB} = 1, x_{BD} = 1$。成本 = 5。

**手算验证**:两条路径:A→B→D = 2+3 = 5,A→C→D = 5+1 = 6。前者短。✓

**Q2.** 路径上的可靠性为 0.9, 0.95, 0.8。对数成本?

$c = -\log r$: 0.105, 0.051, 0.223。总和 = 0.379。可靠性 = $e^{-0.379} = 0.685$。

**核对**:直接乘 $0.9 \cdot 0.95 \cdot 0.8 = 0.684$ ≈ 0.685 ✓(小数点误差)。

**Q3.** "如果路径同时使用 A→C 和 C→E,则必须使用 J→L。"约束?

$x_{A,C} + x_{C,E} - 1 \le x_{J,L}$。

**Q4.** P→LA 分段: 前 50 单位 \$2,51-100 单位 \$4,超过 \$5。请公式化。

添加变量 $x_1$ (≤50, \$2), $x_2$ (≤50, \$4), $x_3$ (无限制, \$5)。守恒中使用 $x_1 + x_2 + x_3$。目标函数加入 $2x_1 + 4x_2 + 5x_3$。

**关键技巧**:三层成本拆三个变量,各层一个上限(最后一层无上限)。LP 自动按"\$2 → \$4 → \$5"顺序使用——因为最小化,贪婪。

**Q5.** 边 (P1,A1) 上的容量 ≤ 160 影子价格为 −8。含义?

每增加一单位容量可节省 \$8 的成本。该边是瓶颈。

**这就是第一周影子价格的应用**:容量约束 $x \le 160$ 的 $\pi = -8$,扩容 1 单位让最小化目标(总成本)减 8。

**Q6.** "不能同时使用边 B→F 和 J→K。"约束?

$x_{B,F} + x_{J,K} \le 1$。

**警告:此约束破坏 TU**——必须把 $x$ 声明为二元,否则 LP 给分数解。

## 15. 速查卡

| 概念 | 公式 |
|---|---|
| 流守恒 | $\sum_j x_{ji} + b_i = \sum_j x_{ij}$ |
| MCNF 目标函数 | $\min \sum c_{ij} x_{ij}$ |
| 供应/需求 | $b_i > 0$ 供应; $< 0$ 需求; $= 0$ 转运点 |
| 平衡 | $\sum b_i = 0$ |
| 最短路径设置 | $b_s = +1, b_t = -1$ |
| 最大可靠性 | $c_{ij} = -\log r_{ij}$,最小化 |
| 前K条路径 | 迭代: $\sum_{(i,j)\in E^*_k} x_{ij} \le L_k - 1$ |
| TU 失效 | 非流守恒约束 → 需要 IP |

**关键提醒:** 当题目说"不使用 Σ 表示法"时,明确列出每个变量。

**分段模板:** 便宜层级变量(有上限)、昂贵层级变量(无上限),两者都在守恒中求和,目标函数中分别列出成本项。

**最大可靠性记住:** 小于 1 的可靠性的对数为负。最大 = 最不负。

## 🎙️ 课堂录音见解 —— 超越幻灯片

### 考试重点(直接引用)

- 网络流和最短路径会**重点测试**。预计有多个题目。
- **3 小时考试。** 把握节奏。
- 重点理解公式化,而不是死记代码。从头编写约束,或在部分公式中找出缺失的约束。
- 练习题包括去年的 5 道 + 4 道新题。有些题特意更难,以利于学习。
- **考试经常问:"你将如何修改这个公式以考虑 [X]?"** 而不是从头开始。

**"修改"型题目的应对**:看到"现在加上 [新条件],修改你的 LP"——不要全部重写,只**加 1-2 条约束**或**改 1 个变量类型**。考试期望你看出最小改动,不是大改造。

### 口头澄清

**流守恒 —— 两种解释:**
1. 逻辑层面:决策变量 = 边是否在路径上(二进制)。
2. 物理层面:变量 = 流过节点的流量单位(流体/产品)。

> *"这种双重视角阐明了为什么去除二进制约束仍然产生整数解。"*

**全幺模性的脆弱性:**

> *"网络流约束矩阵具有全幺模性: 每个方阵子矩阵的行列式 ∈ {−1, 0, +1}。这意味着: 如果放松二进制, LP 会自动给出整数解。"*

**但这个性质很脆弱。** 添加一个非流守恒约束(例如"边 A 或边 B 但不能两者都用")→ LP 返回分数解(例如 0.5)。

### 设置策略(口头分步骤)

1. **识别网络:** 节点(城市)、边(道路)、权重(时间)。
2. **定义决策变量:** 二进制 $x_{ij}$ = 1 如果边在路径上。
3. **起点处的流量:** 恰好一条出边。
4. **终点处的流量:** 恰好一条入边。
5. **中间节点处的流量:** 入边数 = 出边数。
6. **目标函数:** 最小化 $\sum$ 权重 $\times x$。

### 常见错误(口头标注)

**缺失流量平衡** —— 一个常见错误:
- 错: 中间节点 B 写 $x_{BD} + x_{BF} = 1$(无论是否进入 B 都强制有一条出边)。
- 对: $x_{AB} = x_{BD} + x_{BF}$(只有从 A 进入 B 时才从 B 出去)。

**为什么这个错误这么常见**:乍一看 $x_{BD} + x_{BF} = 1$ 也是"B 出去一条边",很有道理——但它**永远强制 B 出一条边**,即使路径根本没经过 B。结果:LP 给一个 disconnected 的路径(A 走了一圈,B 单独又走一段),不合理。

**条件与无条件:**
- $x_{BD} + x_{BF} = x_{AB}$ 是条件性的: "如果我进入 B(即 $x_{AB}=1$),则我恰好从一条边出去。"
- $x_{AB} \cdot (x_{BD} + x_{BF}) = x_{AB}$ 是非线性 → Gurobi 无法求解。

**求和指标不明确:** 嵌套求和时,使用不同的符号($t$ vs $t'$)。

### 实例 —— 最短路径 A→K

网络: 11 个节点(A–K),带权边。学生手动找到:
- 朴素猜测: A→B→E→H→I→K = 12 小时
- **最优解(LP 求解):** A→B→F→J→K = **9 小时**

> *"通过观察并不明显。你需要 LP 来保证最优性。"*

**演示:** 笔记本以 LP 求解(无二进制)。解自动以二进制返回(TU 魔力的证明)。

**然后添加了非网络约束** "$x_{BF} + x_{JK} \le 1$" → LP 返回 $x_{BF} = 0.5$。TU 被破坏。现在需要明确的二进制。

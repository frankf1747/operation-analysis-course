# 第七周 — 网络流问题

## 0. 总体概览

包裹运输、电网、血液供应链 —— 这些都是由节点通过边连接而成的网络。**网络流问题** 优化什么、多少流量、流向何处。**神奇之处**：它们的 LP 松弛总是有整数最优解（**全幺模性**）。所以我们以 LP 求解,却能免费获得整数答案。

本周统一了：**最短路径** (最短路径)、**最小成本网络流** (最小成本网络流) 和 **运输问题**,以及带可靠性和附加约束的变体。

## 1. 图论基础

- **节点**（顶点, 节点）：城市、仓库、交叉点
- **边**（弧, 边）：有向连接（A → B 表示流量可以从 A 流向 B,但反向除非有单独的边）
- **源**（源）：具有供应的节点（产生流量）
- **汇**（汇）：具有需求的节点（消耗流量）
- **转运点**（转运点）：无供应或需求；流量经过
- **容量** $u_{ij}$：边上的最大流量
- **成本** $c_{ij}$：每单位运输成本

## 2. 最小成本网络流 (MCNF)

### 设置
- 有向图 $(N, E)$
- 边 $(i,j)$：成本 $c_{ij}$,容量 $u_{ij}$
- 节点 $i$：供应 $b_i$ ($> 0$ = 供应, $< 0$ = 需求, $= 0$ = 转运点)
- **平衡条件: $\sum_i b_i = 0$**

### 决策变量
$x_{ij}$ = 边 $(i,j)$ 上的流量。

### 流守恒 流守恒 (关键约束)
在每个节点 $i$:
$$\sum_{j: (j,i) \in E} x_{ji} + b_i = \sum_{j: (i,j) \in E} x_{ij}$$
**流入 + 供应 = 流出 + 需求**

等价地：**净流出 = 供应**。

### 完整 LP
$$\min \sum_{(i,j) \in E} c_{ij} x_{ij}$$
$$\text{s.t. 每个节点的流守恒}$$
$$0 \le x_{ij} \le u_{ij}$$

## 3. 全幺模性 完全幺模性

**定理：** 如果约束矩阵是 TU 且所有右端项为整数,则每个基本可行解都是整数。

**含义：** 纯网络流 LP → 最优 $x_{ij}$ 自动为整数。无需分支定界。

**失效条件：** 添加非流守恒约束（例如, $x_{B,F} + x_{J,K} \le 1$）。此时需要 IP。

## 4. 运输问题（特殊 MCNF）

- 二部图：源 → 汇 直接相连（无转运点）
- $m$ 个供应节点, $n$ 个需求节点
- $m \times n$ 条边
- 特殊情况可由专门算法求解（西北角法、Vogel 法）

## 5. 最短路径作为 MCNF

求 $s \to t$ 的最小成本路径：
- $b_s = +1$, $b_t = -1$,其他 $= 0$
- $u_{ij} = 1$（或 $\infty$ —— LP 会选择二进制）
- 最小化 $\sum c_{ij} x_{ij}$

$x_{ij} = 1$ 的边构成最短路径。

## 6. 最短路径笔记本 —— 单元格演练

11 个节点,17 条边。源 A,汇 K。

```python
edge_data = [("A","B",3), ("A","C",5), ...]
m = Model()
x = m.addVars(edges, lb=0, ub=1)
# A: must leave
m.addConstr(sum(x["A",j] for j in outgoing(A)) == 1)
# K: must arrive
m.addConstr(sum(x[i,"K"] for i in incoming(K)) == 1)
# Others: balance
for i in middle_nodes:
    m.addConstr(sum(x[i,j] for j) == sum(x[j,i] for j))
m.setObjective(sum(time[i,j]*x[i,j] for (i,j) in edges), GRB.MINIMIZE)
m.optimize()
# Optimal: A → B → F → J → K, cost 9
```

**添加非网络约束：** $x_{B,F} + x_{J,K} \le 1$。LP 现在给出分数解 $x_{B,F} = 0.5$（TU 被破坏）。

## 7. MCNF 笔记本 —— 演练

2 个工厂、5 个仓库、2 个零售商,共 14 个节点。加载 CSV,为每个节点构建流守恒。求解 → 成本 $8,860。

**扩展生产决策：**
- $y_n$ = 工厂 $n$ 的产量（变量）
- $z_n$ = 零售商 $n$ 处的短缺（允许的未满足需求）
- 零售需求增加了 50%（不平衡）
- 最优解: $y_{P1} = 320, y_{P2} = 240, z_{R1} = 100$。成本 $18,360。

**容量约束上的影子价格** 揭示了瓶颈。边 (P1, A1): $\pi = -10$。将容量扩大 50 可节省约 $500。使用二进制 $w_e$ 来选择扩展哪条边。

## 8. 生产 + 运输 练习题（网络流-I）—— 核心例题

### 问题：Psyche 鞋业公司
- **生产工厂：** Dallas（$10/单位）, Albuquerque（$12/单位）
- **配送中心：** Phoenix（库存 100）, Las Vegas（库存 0）
- **零售店：** LA（需求 260）, SF（需求 450）
- **第三方订单：** 200 单位直接从 Dallas 履行
- **LA/SF 以外的在线需求：** 510 单位,从 Las Vegas 履行

**运输成本（$/单位）：**
- D→LV: 5, D→P: 7, A→LV: 4, A→P: 9
- P→LA: 3, LV→LA: 3, LV→SF: 5

### 完整 LP（无 Σ 表示 —— 考试格式）

**变量：** $y_D, y_A$（生产）; $x_{ij}$（每条边上的流量）。

**目标函数：**
$$\min 10y_D + 12y_A + 5x_{D,LV} + 7x_{D,P} + 4x_{A,LV} + 9x_{A,P} + 3x_{P,LA} + 3x_{LV,LA} + 5x_{LV,SF}$$

**约束条件：**
- Dallas: $y_D = 200 + x_{D,LV} + x_{D,P}$
- Albuquerque: $y_A = x_{A,LV} + x_{A,P}$
- Las Vegas: $x_{D,LV} + x_{A,LV} = 510 + x_{LV,LA} + x_{LV,SF}$
- Phoenix: $x_{D,P} + x_{A,P} + 100 = x_{P,LA}$
- LA: $x_{P,LA} + x_{LV,LA} = 260$
- SF: $x_{LV,SF} = 450$
- 所有 $\ge 0$

**数字来源：**
- Dallas 的 200 = 第三方需求
- LV 的 510 = 所有 LA/SF 以外的在线订单总和 (150+200+90+70)
- Phoenix 的 100 = 初始库存
- LA 的 260 = 零售 (200) + 在线 (60)
- SF 的 450 = 零售 (100) + 在线 (350)

## 9. 分段线性成本

"前 100 单位 P→LA 价格 $3,超过部分 $6。"

**修改方式：**
1. 添加新变量 $\tilde x_{P,LA}$（价格 $6 的超额流量）。
2. 限制 $x_{P,LA} \le 100$。
3. 修改 Phoenix: $x_{D,P} + x_{A,P} + 100 = x_{P,LA} + \tilde x_{P,LA}$。
4. 修改 LA: $x_{P,LA} + \tilde x_{P,LA} + x_{LV,LA} = 260$。
5. 在目标函数中加入 $+6 \tilde x_{P,LA}$。

**为何有效：** 最小化成本时,LP 会先把便宜的 $x_{P,LA}$ 填满到 100 的上限,然后才使用 $\tilde x$。

## 10. 最大可靠性路径 —— 练习题 网络流-II

### 问题
管道 A → L。每条边有可靠性 $r_{ij} \in (0, 1)$。路径可靠性 = 各边可靠性的乘积。

### 关键见解：对数变换
$$\max \prod r_{ij} = \max \log \prod r_{ij} = \max \sum \log r_{ij}$$

由于当 $r_{ij} < 1$ 时 $\log r_{ij} < 0$,总和为负。最大化总和 = 最不负的路径。

**等价于：** $\min \sum (-\log r_{ij})$,其中 $-\log r_{ij} > 0$ → 最短路径。

### 公式化
带有边成本 $c_{ij} = -\log r_{ij}$ 的最短路径 IP。

### 动态规划（反向递归）
$v_i$ = 从 $i$ 到 $L$ 的最大对数可靠性。
$v_L = 0$。
$v_i = \max_{j \in O(i)} \{\log r_{ij} + v_j\}$。

**练习题结果：** $v_A = -2.3$。最大可靠性 = $e^{-2.3} = 0.100$。路径: A → C → D → H → J → L。

### 路径上的附加约束
- "如果 C→E 则不能 G→J": $x_{C,E} + x_{G,J} \le 1$
- "如果 A→C 且 C→E 则 J→L": $x_{A,C} + x_{C,E} - 1 \le x_{J,L}$
- "如果经过 E（使用 E→G）, ≥ 4 条边": $4 x_{E,G} \le \sum_{(i,j)} x_{ij}$

### 前K条最短路径（迭代）
1. 求解,获得路径 1 的边集 $E^{*,1}$,长度 $L_1$。
2. 添加: $\sum_{(i,j) \in E^{*,1}} x_{ij} \le L_1 - 1$。
3. 重新求解 → 路径 2。
4. 重复 $k$ 次。

## 11. 分步方法

### (a) 将文字题转化为网络
1. 每个实体 = 节点。
2. 每条运输路线 = 有向边。
3. 供应 (+), 需求 (−), 转运点 (0)。
4. 每条边的成本和容量。

### (b) 明确写出流守恒
对每个节点 $i$: $\sum_j x_{ji} + b_i = \sum_j x_{ij}$。列出所有变量。

### (c) 分段成本
将边拆分为各层级变量；对每个限制上限；在守恒中求和。

### (d) 最短路径
$b_s = 1, b_t = -1$,其他为 0; 最小化 $\sum c_{ij} x_{ij}$。

### (e) 最大可靠性
与最短路径相同,但 $c_{ij} = -\log r_{ij}$。

### (f) 路径附加约束
关于 $x_{ij}$ 的线性约束（破坏 TU；可能需要 IP）。

## 12. 常见错误 ⚠️

1. **流守恒符号错误。** 净流出 = 供应。
2. **忘记初始库存。** Phoenix 手头有 100；包括它。
3. **混淆平衡与非平衡。** $\sum b_i$ 必须等于 0（或具有短缺/过剩变量）。
4. **边的方向错误。** A → B 是有向的。
5. **忘记非负性。**
6. **对数可靠性符号混淆。** 对数是负的；最小化负值。
7. **没有二进制就添加非网络约束。** 破坏 TU。
8. **不理解为何分段有效。** LP 贪婪地使用更便宜的选项。

## 14. 小练习

**Q1.** 节点 {A,B,C,D}。边: A→B(2), A→C(5), B→D(3), C→D(1)。最短 A→D 路径？

$b_A = 1, b_D = -1, b_B = b_C = 0$。最优解: $x_{AB} = 1, x_{BD} = 1$。成本 = 5。

**Q2.** 路径上的可靠性为 0.9, 0.95, 0.8。对数成本？

$c = -\log r$: 0.105, 0.051, 0.223。总和 = 0.379。可靠性 = $e^{-0.379} = 0.685$。

**Q3.** "如果路径同时使用 A→C 和 C→E,则必须使用 J→L。"约束？

$x_{A,C} + x_{C,E} - 1 \le x_{J,L}$。

**Q4.** P→LA 分段: 前 50 单位 $2,51-100 单位 $4,超过 $5。请公式化。

添加变量 $x_1$ (≤50, $2), $x_2$ (≤50, $4), $x_3$ (无限制, $5)。守恒中使用 $x_1 + x_2 + x_3$。目标函数加入 $2x_1 + 4x_2 + 5x_3$。

**Q5.** 边 (P1,A1) 上的容量 ≤ 160 影子价格为 −8。含义？

每增加一单位容量可节省 $8 的成本。该边是瓶颈。

**Q6.** "不能同时使用边 B→F 和 J→K。"约束？

$x_{B,F} + x_{J,K} \le 1$。

## 15. 速查卡

| 概念 | 公式 |
|---|---|
| 流守恒 | $\sum_j x_{ji} + b_i = \sum_j x_{ij}$ |
| MCNF 目标函数 | $\min \sum c_{ij} x_{ij}$ |
| 供应/需求 | $b_i > 0$ 供应; $< 0$ 需求; $= 0$ 转运点 |
| 平衡 | $\sum b_i = 0$ |
| 最短路径设置 | $b_s = +1, b_t = -1$ |
| 最大可靠性 | $c_{ij} = -\log r_{ij}$,最小化 |
| 前K条路径 | 迭代: $\sum_{(i,j)\in E^*_k} x_{ij} \le L_k - 1$ |
| TU 失效 | 非流守恒约束 → 需要 IP |

**关键提醒：** 当题目说"不使用 Σ 表示法"时,明确列出每个变量。

**分段模板：** 便宜层级变量（有上限）、昂贵层级变量（无上限）,两者都在守恒中求和,目标函数中分别列出成本项。

**最大可靠性记住：** 小于 1 的可靠性的对数为负。最大 = 最不负。

### 口头澄清

**流守恒 —— 两种解释：**
1. 逻辑层面：决策变量 = 边是否在路径上（二进制）。
2. 物理层面：变量 = 流过节点的流量单位（流体/产品）。

> *"这种双重视角阐明了为什么去除二进制约束仍然产生整数解。"*

**全幺模性的脆弱性：**

> *"网络流约束矩阵具有全幺模性: 每个方阵子矩阵的行列式 ∈ {−1, 0, +1}。这意味着: 如果放松二进制, LP 会自动给出整数解。"*

**但这个性质很脆弱。** 添加一个非流守恒约束（例如"边 A 或边 B 但不能两者都用"）→ LP 返回分数解（例如 0.5）。

### 设置策略（口头分步骤）

1. **识别网络：** 节点（城市）、边（道路）、权重（时间）。
2. **定义决策变量：** 二进制 $x_{ij}$ = 1 如果边在路径上。
3. **起点处的流量：** 恰好一条出边。
4. **终点处的流量：** 恰好一条入边。
5. **中间节点处的流量：** 入边数 = 出边数。
6. **目标函数：** 最小化 $\sum$ 权重 $\times x$。

### 常见错误（口头标注）

**缺失流量平衡** —— 一个常见错误：
- 错: 中间节点 B 写 $x_{BD} + x_{BF} = 1$（无论是否进入 B 都强制有一条出边）。
- 对: $x_{AB} = x_{BD} + x_{BF}$（只有从 A 进入 B 时才从 B 出去）。

**条件与无条件：**
- $x_{BD} + x_{BF} = x_{AB}$ 是条件性的: "如果我进入 B（即 $x_{AB}=1$）,则我恰好从一条边出去。"
- $x_{AB} \cdot (x_{BD} + x_{BF}) = x_{AB}$ 是非线性 → Gurobi 无法求解。

**求和指标不明确：** 嵌套求和时,使用不同的符号（$t$ vs $t'$）。

### 实例 —— 最短路径 A→K

网络: 11 个节点（A–K）,带权边。学生手动找到：
- 朴素猜测: A→B→E→H→I→K = 12 小时
- **最优解（LP 求解）:** A→B→F→J→K = **9 小时**

> *"通过观察并不明显。你需要 LP 来保证最优性。"*

**演示：** 笔记本以 LP 求解（无二进制）。解自动以二进制返回（TU 魔力的证明）。

**然后添加了非网络约束** "$x_{BF} + x_{JK} \le 1$" → LP 返回 $x_{BF} = 0.5$。TU 被破坏。现在需要明确的二进制。

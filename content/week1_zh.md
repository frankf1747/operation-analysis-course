# 第一周 — 线性规划对偶

## 0. 总体概览

设想你拥有一家面包店,想通过决定烘焙什么产品来最大化利润。你知道你的原料限制:26kg 黄油、80kg 面粉、5kg 糖。

但还有一个更难的问题:**每种资源对你的业务实际上值多少钱?**

举个例子,如果今天有人愿意以每克 \$0.02 卖给你更多面粉,你应该买吗?是 \$0.005 呢?是 \$0.10 呢?这个问题表面上是市场价问题,实际上是问:**多一克面粉,能让我多赚多少?**

**对偶 (duality) 就是用来回答这种问题的。**

对偶把你的问题反转过来:
- **原问题 (primal)** 问:"我应该生产多少 bun、croissant、muffin 来最大化收入?"
- **对偶问题 (dual)** 问:"如果我在资源市场上竞争,每种原材料应该值多少钱?"

原问题告诉你**最优的生产计划**;对偶问题告诉你**每种受约束资源的隐含价值**——也就是大名鼎鼎的 **影子价格 (shadow price)**。

这不是纯理论。这些影子价格让你能够就"是否购买更多投入"做出真实的商业决策。而且——这是最妙的地方——它们**无需额外求解原问题 LP** 即可获得。Gurobi 求一次 LP,同时给你两边的答案。

**强对偶 (strong duality)** 是整周最重要的事实:原问题的最优利润 = 对偶问题的最优资源估值成本。它们是同一枚硬币的两面。如果两边的目标值能对上,你就同时知道两件事——最优生产计划是什么,以及瓶颈资源在边际上值多少钱。

本周教你三件事:
1. 对偶的**机制**——给定一个 LP,怎么写出它的对偶问题。
2. 影子价格的**解读**——拿到对偶解之后,它在告诉你什么经济含义。
3. **灵敏度分析 (sensitivity analysis)**——用影子价格回答关于运营的"如果……怎么办"问题。

## 1. 线性规划快速复习(给没准备的同学)

**线性规划 (LP)** 是一种优化问题,即在线性约束下最大化(或最小化)一个线性目标函数。

为什么叫"线性"?因为目标函数和约束都是变量的线性组合——没有平方项、没有 log、没有变量乘变量。这个限制看起来很严,但带来了巨大的好处:LP **总是有有限步骤的算法**(单纯形、内点)能求出全局最优解,而且我们能写出"对偶"。一旦你加了非线性,这些好处就全没了。

**标准形式(最大化问题):**
$$\text{maximize } \mathbf{c}^T \mathbf{x}$$
$$\text{subject to } A\mathbf{x} \le \mathbf{b}$$
$$\mathbf{x} \ge \mathbf{0}$$

**关键术语:**
- **决策变量** ($\mathbf{x}$):你要决定什么(生产多少、分配多少等)
- **目标函数** ($\mathbf{c}^T \mathbf{x}$):你要优化的利润、成本或收入
- **约束** ($A\mathbf{x} \le \mathbf{b}$):资源限制、需求等
- **可行域**:所有满足全部约束的 $\mathbf{x}$ 的集合
- **最优解**:具有最佳目标值的可行点

这五个术语后面会反复出现,所以混熟它们。特别注意:**约束**永远是不等式或等式,**目标函数**永远只有一个,**决策变量**才是你"控制"的东西。

**两变量小例子:**
$$\text{maximize } 3x_1 + 2x_2 \quad \text{s.t.} \quad x_1 + x_2 \le 4,\; x_1, x_2 \ge 0$$

从几何上看,可行域是一个角点位于 $(0,0)$、$(4,0)$ 和 $(0,4)$ 的三角形。目标函数 $3x_1 + 2x_2$ 定义了一族平行线。你将这些线向外滑动,直到它们恰好在一个角点处接触可行域——这里是 $(4,0)$,目标值为 $12$。该角点就是最优解。

**为什么最优解永远在角点?** 因为目标函数是线性的,而可行域是凸多边形——你沿着任何方向走,目标值要么单调上升要么单调下降。所以"最远的"那个角点必然是最优。LP 的整个简单性来自这一句话:**只需要检查有限多个角点,而不是检查整个可行域**。

## 2. 逐张幻灯片讲解

### 引言与背景
来自 MGMTMSA 403 的 LP 复习,进阶到:(1) 什么是对偶问题,(2) 什么是影子价格,(3) 灵敏度分析。工具:Python + Gurobi + Jupyter。

### 面包店生产问题

一家面包店每周生产三种产品:小圆面包 (bun)、可颂 (croissant) 和松饼 (muffin)。

| 产品 | 单价 | 黄油 (g/单位) | 面粉 (g/单位) | 糖 (g/单位) |
|---------|------------|-----------------|----------------|----------------|
| Bun     | \$3.00      | 20              | 100            | 0              |
| Croissant | \$4.00    | 100             | 50             | 50             |
| Muffin  | \$4.50      | 10              | 0              | 150            |

可用量:26,000g 黄油、80,000g 面粉、5,000g 糖。

**读懂这张表的方法:** 横着看,告诉你"做一个 X 要消耗多少资源,卖多少钱";竖着看,告诉你"某种资源同时被哪些产品争抢"。后者特别重要,因为对偶的核心就是问:"这种被争抢的资源,边际上值多少钱?"

注意一个有趣的事实——muffin 卖最贵 (\$4.50),但要 150g 糖,而我们只有 5,000g 糖。也就是说,糖只够做 33 个 muffin 就用完了。所以"muffin 卖最贵"不代表"应该多做 muffin"——糖会先卡住你。这就是 LP 的价值:它会自动平衡这些 tradeoff。

**原问题 LP:**
$$\max 3x_B + 4x_C + 4.5x_M$$
$$\text{s.t.}\; 20x_B + 100x_C + 10x_M \le 26000 \text{ (butter)}$$
$$100x_B + 50x_C + 0x_M \le 80000 \text{ (flour)}$$
$$0x_B + 50x_C + 150x_M \le 5000 \text{ (sugar)}$$
$$x_B, x_C, x_M \ge 0$$

这是一个配方平衡问题。每种产品需要一定量的三种原料。我们的库存有限,希望最大化总销售价值。

**为什么是 ≤ 而不是 =?** 因为我们不必把所有原料用完——用 1g 黄油也行,用 26000g 也行,只要不超出库存。把它写成 = 会人为缩小可行域,通常导致更差的解。

**为什么是 $x \ge 0$?** 生产 −5 个 bun 没有现实意义。这个约束叫**非负性约束**,几乎所有商业 LP 都有它。

### 为我们的 LP 设定上界(对偶的动机)

"在不求解 LP 的情况下,你能否得到最佳可能收入的上界?"

为什么我们关心上界?设想你猜测最优收入是 \$2,650。如果有人告诉你"无论怎么做,你的最大收入都不可能超过 \$2,650",那你立刻就知道 \$2,650 已经是最优了——**不需要枚举所有可能的生产计划**。上界就是这种"封顶"工具。

用权重 $p_B$、$p_F$、$p_S \ge 0$ 组合这三个约束:
$$p_B \cdot (\text{butter constraint}) + p_F \cdot (\text{flour constraint}) + p_S \cdot (\text{sugar constraint})$$

如果我们选择使左侧系数主导目标的权重:
- $20p_B + 100p_F + 0p_S \ge 3$
- $100p_B + 50p_F + 50p_S \ge 4$
- $10p_B + 0p_F + 150p_S \ge 4.5$

则对任何可行的 $x$:$3x_B + 4x_C + 4.5x_M \le 26000 p_B + 80000 p_F + 5000 p_S$。

**这一步的代数逻辑:** 左边每一项 $c_j x_j$,因为约束系数 $\ge c_j$,所以可以放大成 $\sum_i p_i a_{ij} x_j$。把所有 $j$ 加起来,左边是原问题的目标 $c^T x$,右边是 $p^T A x \le p^T b$(因为 $A x \le b$ 且 $p \ge 0$)。结论:$c^T x \le p^T b$,无论 $x$ 是什么。

**为什么用这些权重?** 它们代表资源的隐含价格。

把 $p_B, p_F, p_S$ 想成"如果我以 $p_B$ 元/克卖掉所有黄油,以 $p_F$ 元/克卖掉所有面粉,以 $p_S$ 元/克卖掉所有糖,我能换多少钱?"——也就是 $26000 p_B + 80000 p_F + 5000 p_S$。这是把整个工厂"作价拍卖"的收益。

约束 $20p_B + 100p_F + 0p_S \ge 3$ 的意思是:**生产一个 bun 所消耗的资源,按这个价格算,至少要值 3 块**——否则你直接卖原料就比做 bun 划算。三个约束分别对应三个产品。

### 弱对偶 (Weak Duality)

**弱对偶定理:** 对任何可行的原问题解和任何可行的对偶问题解:
$$\text{primal objective} \le \text{dual objective}$$

直观地说:任何"可行的生产计划带来的收入" ≤ 任何"可行的资源价格带来的总价值"。最优生产计划的收入,被任何对偶可行解给"封顶"了。

这就是为什么对偶问题叫"对偶"——它从外侧把原问题夹住。原问题最大化收入(从下面顶起),对偶问题最小化估值(从上面压下),两者朝同一个值靠近。

如果我们生产 ($x_B = 100, x_C = 50, x_M = 0$),收入为 $700$,并猜测价格 $(p_B, p_F, p_S) = (0.02, 0.01, 0.05)$,则对偶问题的目标为:
$$0.02 \cdot 26000 + 0.01 \cdot 80000 + 0.05 \cdot 5000 = 1570$$
弱对偶:$700 \le 1570$ ✓。

这两个数 $700$ 和 $1570$ 之间的空隙,叫**duality gap**。空隙意味着:要么生产计划不是最优的,要么对偶价格不是最优的,要么两者都不是。最优时,这个空隙必须等于 0(就是后面的强对偶)。

这给了我们一种验证最优性的方法:如果 原问题 = 对偶问题,两者都是最优的。

**这一招在考试里很有用。** 给你一对 $(x, p)$,问是不是最优?
1. 检查 $x$ 原问题可行,$p$ 对偶可行。
2. 算两个目标值。
3. 如果相等 → 都是最优,不需要再做别的。
4. 如果不等 → 至少有一个不是最优。

### 机械地构造对偶问题

对我们的**最大化原问题**(3 个 ≤ 约束,3 个 ≥0 变量),**对偶问题是最小化**:

**对偶问题:**
$$\min 26000p_B + 80000p_F + 5000p_S$$
$$\text{s.t.}\; 20p_B + 100p_F + 0p_S \ge 3 \text{ (bun's coef)}$$
$$100p_B + 50p_F + 50p_S \ge 4 \text{ (croissant's coef)}$$
$$10p_B + 0p_F + 150p_S \ge 4.5 \text{ (muffin's coef)}$$
$$p_B, p_F, p_S \ge 0$$

对偶问题中每个原问题约束对应一个变量,每个原问题变量对应一个约束。对偶目标函数为 RHS × 价格之和;对偶约束确保价格"高到足以"覆盖产品的获利能力。

**怎么对应起来——一步步看:**
- 原问题有 **3 个资源约束** (butter, flour, sugar) → 对偶有 **3 个变量** ($p_B, p_F, p_S$),每个变量是对应资源的"价格"。
- 原问题有 **3 个产品决策变量** ($x_B, x_C, x_M$) → 对偶有 **3 个约束**,每个约束对应一种产品。
- **目标函数翻转方向**:原问题 max → 对偶 min。
- **目标系数和 RHS 互换**:原问题的资源上限 (26000, 80000, 5000) 变成对偶的目标系数;原问题的产品单价 (3, 4, 4.5) 变成对偶约束的 RHS。
- **约束矩阵转置**:原问题约束矩阵 $A$ 的转置成为对偶约束矩阵。bun 在原问题里用 (20, 100, 0) 三种资源 → 在对偶里,bun 的约束系数就是 (20, 100, 0),对应 ($p_B, p_F, p_S$)。

**直觉记法**:对偶问题就是把原问题的表格"转过来"看。横和竖换一下,max 和 min 换一下,目标和 RHS 换一下,约束方向也跟着翻。

### 强对偶 (Strong Duality)

**强对偶定理:** 如果原问题 LP 有最优解,则对偶问题 LP 也有最优解,并且两者的最优目标值相等。

**面包店例子:**
- 原问题最优:$x_B = 750, x_C = 100, x_M = 0$ → 收入 = $3(750) + 4(100) = 2650$
- 对偶问题最优:$p_F = 0.03, p_S = 0.05, p_B = 0$(来自 notebook)→ 由强对偶,成本与 2650 相等

**验算一下对偶目标:** $26000 \cdot 0 + 80000 \cdot 0.03 + 5000 \cdot 0.05 = 0 + 2400 + 250 = 2650$ ✓

完全匹配。这不是巧合——这就是强对偶在保证的事情。

**为什么强对偶很重要(实务上的意义)?** 因为它给了你**最简单的最优性证明**。在考试里,如果你算出原问题 = \$2,650,对偶 = \$2,650,两者都可行,你立刻知道你找到了最优——**不需要互补松弛,不需要灵敏度检查,什么都不需要**。教授在课上反复强调:"强对偶是最简单的最优性检验。" 牢记这句话。

**警告**:强对偶**只对 LP 成立**。一旦你加了整数约束 (IP) 或非线性,强对偶就破了——你只能得到所谓"拉格朗日对偶",它给你一个上下界,但不是精确值。这就是为什么影子价格不能用在整数规划里。

### 影子价格 (Shadow Prices)

约束 $i$ 对应的最优对偶变量 $p_i$ 即为**影子价格**。它回答:

> "如果我把约束 $i$ 放松一个单位,最优目标值会增加多少?"

换句话说,$p_i = \frac{\partial (\text{最优目标值})}{\partial b_i}$——它是目标值对该约束 RHS 的**导数**。用泰勒展开的语言:
$$f(b_i + h) \approx f(b_i) + p_i \cdot h$$
小幅增加 $h$ 单位资源 $i$,目标值大约增加 $p_i \cdot h$。

**面包店解读:**
- 面粉:$p_F = 0.03$/g → 多 1 克面粉增加 \$0.03 收入
- 糖:$p_S = 0.05$/g → 多 1 克糖增加 \$0.05 收入
- 黄油:$p_B = 0$/g → 黄油不是瓶颈(我们还剩 1000g 未用)

**怎么读这三个数:**
1. $p_F = 0.03$ 意味着面粉是**瓶颈**——多给 1g 面粉,我能多赚 3 分。
2. $p_S = 0.05$ 意味着糖是**最稀缺**的——多 1g 糖能多赚 5 分,比面粉的边际价值还高。
3. $p_B = 0$ 意味着黄油**根本不是瓶颈**——多给我一克我也用不上,所以它的边际价值是 0。

这就是影子价格的全部精髓。**瓶颈资源的影子价格 > 0,有剩余的资源影子价格 = 0**。后面互补松弛会把这件事正式化。

**决策规则:** 当市场价格 < 影子价格时,购买资源。如果面粉价格为 \$0.02/g 而影子价格为 \$0.03/g,则购买(每克净增益 \$0.01)。

更一般地,这条规则是:**比较"市场买入价 $q$"和"边际收益 $p^*$"**:
- $q < p^*$ → **买**(净赚 $p^* - q$ 每单位)
- $q > p^*$ → **不买**(净亏)
- $q = p^*$ → **无差异**(边界)

**注意单位!** 影子价格 \$0.03/g 和市场价 \$5/kg 是不一样的单位——\$5/kg = \$0.005/g。**比较前一定换算**,这是考试最常见的坑。

**警告**:这条规则只在**小幅变化**时成立。如果有人卖你 50,000g 面粉,影子价格不再准确,因为另一个约束(比如糖)会成为新的瓶颈。Gurobi 用 `.SARHSLow` 和 `.SARHSHigh` 报告每个影子价格的"有效范围"。

### 互补松弛 (Complementary Slackness)

互补松弛 (CS) 是把"最优时谁是瓶颈、谁是垃圾"用代数表达出来。它有**两组**对称的关系——分别看会清楚很多。

#### 第一组:如果某个产品被生产了 ($x_j > 0$),那它对应的对偶约束必须是紧的

形式上:$x_j^* > 0$ $\Rightarrow$ 对偶约束 $j$ 取等号。

**举例**:在面包店里,$x_B = 750 > 0$,意思是我们真的在生产 bun。既然 bun 被生产了,那么 bun 的"收入"必须**刚好等于**它消耗资源的隐含价值。

Bun 的对偶约束是:
$$20p_B + 100p_F + 0p_S \ge 3$$

最优时这个不等式必须取等号:
$$20p_B + 100p_F + 0p_S = 3$$

代入最优对偶解 $(p_B, p_F, p_S) = (0, 0.03, 0.05)$:
$$20(0) + 100(0.03) + 0(0.05) = 3 \;\;\checkmark$$

**经济含义**:一个 bun 带来 \$3 收入 = 一个 bun 消耗资源的隐含价值 (\$3)。盈亏完全打平。

**为什么必须打平?** 反证一下:
- 如果"消耗的资源价值" > 3,那做 bun 是赔本——理性的厂家不会做,即 $x_B = 0$。矛盾。
- 如果"消耗的资源价值" < 3,那做 bun 赚得太多,LP 还没收敛到最优——应该继续多做 bun。矛盾。

所以最优时,被生产的产品必须"价格 = 隐含成本"。这就是 CS 第一组的全部内容。

**反之**,如果一个产品**没有被生产** ($x_j = 0$),它的对偶约束可以**有松弛**——意思是"做这个产品成本太高了,不值得做"。在面包店里,$x_M = 0$,muffin 没做,因为它的对偶约束 $10(0) + 0(0.03) + 150(0.05) = 7.5 > 4.5$——成本比收入高得多。

#### 第二组:如果某个资源有剩余 (有 slack),那它的影子价格必须是 0

形式上:**原问题约束 $i$ 有松弛 (LHS < RHS)** $\;\Rightarrow\; p_i^* = 0$。

**举例**:在面包店里,黄油的最优用量是 $20(750) + 100(100) + 10(0) = 25000$ g,但库存有 $26000$ g——剩余 1000g。所以:
$$\text{butter slack} = 26000 - 25000 = 1000 > 0$$

按 CS 第二组,黄油的影子价格必须为 0:
$$p_B = 0 \;\;\checkmark$$

**经济含义**:多 1g 黄油不会提高最优收入,所以黄油的 shadow price 是 0。

**这非常直观**——如果一种资源都没用完,那你再多给我一点也没用,所以它的边际价值就是 0。

#### 反方向也成立(很多学生忘记)

CS 不止单向,它是**双向**的:
- 松弛 > 0 $\;\Rightarrow\;$ 对偶变量 = 0
- 对偶变量 > 0 $\;\Rightarrow\;$ 约束是紧的

第二个方向的意思是:如果你看到 $p_F = 0.03 > 0$,你立刻知道**面粉约束必须取等号**——也就是面粉一定被用光了。这给你一个强有力的"反向推断"工具。

#### 完整的面包店验证

| 关系 | 检查 | 结论 |
|---|---|---|
| $x_B = 750 > 0$ | bun 对偶约束 $= 20(0) + 100(0.03) + 0(0.05) = 3$ | 紧 ✓ |
| $x_C = 100 > 0$ | croissant 对偶约束 $= 100(0) + 50(0.03) + 50(0.05) = 4$ | 紧 ✓ |
| $x_M = 0$ | muffin 对偶约束 $= 7.5 > 4.5$ | 松弛,允许 ✓ |
| 黄油松弛 = 1000 | $p_B = 0$ | 一致 ✓ |
| 面粉松弛 = 0 | $p_F = 0.03 > 0$ 允许 | 一致 ✓ |
| 糖松弛 = 0 | $p_S = 0.05 > 0$ 允许 | 一致 ✓ |

**用一句话总结 CS**:**互补松弛 = 经济直觉的代数化身**。它说的就是"被生产的产品必须刚好打平,有剩余的资源必须不值钱"。如果你能把这一句话背下来,CS 在考试里你就不会出错。

## 3. 面包店例子 — 完整数值演算

上面已经覆盖。关键事实:
- 原问题最优 = 2650,其中 $(x_B, x_C, x_M) = (750, 100, 0)$
- 对偶问题最优 = 2650,其中 $(p_B, p_F, p_S) = (0, 0.03, 0.05)$(近似值)
- 松弛:黄油 1000g,面粉 0,糖 0

把这三行牢牢记住——它们是后面所有数值检查的"地标"。考试题里如果出现面包店变种,你立刻可以对照。

**灵敏度分析:** 是否以 \$0.005/g 购买面粉?是的——每克净增益 0.025。

**逐步验算这个决策:**
1. 影子价格 $p_F = 0.03$/g(面粉的边际价值)。
2. 市场价 $q = 0.005$/g(对方报价)。
3. 比较 $q$ vs $p_F$:$0.005 < 0.03$ → 买。
4. 每克净增益 = $p_F - q = 0.03 - 0.005 = 0.025$。

**注意**:如果对方报价是 \$5/kg = \$0.005/g,数字一样——但**单位换算是关键**。先把所有价格换到同一单位再比。

## 4. 面包店 LP Notebook — 单元格讲解

**Cell 1: 导入**
```python

```

`from ... import *` 把 Gurobi 的 `Model`、`GRB`、`addVar` 等全部"裸"引进来。生产代码里别这么写(命名空间污染),但课堂例子里方便。

**Cell 3: 创建模型、变量、约束、目标函数**
```python
m = Model()

x_C = m.addVar()
x_M = m.addVar()
```

**Cell 5: 求解**


**Cell 10: 提取解**

- `.X` = 最优变量值,`.objval` = 最优目标值。

**Cell 13: 显式求解对偶问题**

```python
m_dual = Model()

m_dual.addConstr(20*p_B + 100*p_F + 0*p_S >= 3)
```

## 5. 原问题/对偶问题转换规则

| 原问题 (min) | 对偶问题 | (变量符号) |
|--------|--------|--------|
| 约束 ≥ $b_i$ | $p_i \ge 0$ | |
| 约束 ≤ $b_i$ | $p_i \le 0$ | |
| 约束 = $b_i$ | $p_i$ 自由 | |
| 变量 $x_j \ge 0$ | 对偶约束 $p^T A_j \le c_j$ | |
| 变量 $x_j \le 0$ | 对偶约束 $p^T A_j \ge c_j$ | |
| 变量 $x_j$ 自由 | 对偶约束 $p^T A_j = c_j$ | |

**对最大化原问题:** 从右向左读表(符号翻转)。

**怎么用这张表(机械流程):**
1. 先**声明**你的原问题方向(min 还是 max)——错的方向直接全错。
2. 对每个原问题**约束**,查表第 1–3 行,确定对应的对偶变量符号。
3. 对每个原问题**变量**,查表第 4–6 行,确定对应的对偶约束方向。
4. 目标:原问题 RHS → 对偶目标系数,原问题目标系数 → 对偶约束 RHS。
5. 翻转目标:max ↔ min。

### 每条规则的存在原因(直觉)

- **min 原问题中的 ≤ → $p_i \le 0$:** 放松 ≤ 约束(增加 RHS)使可行域变大,只会减小最小化目标值。所以影子价格是非正的。
- **min 原问题中的 ≥ → $p_i \ge 0$:** 放松 ≥ 约束使可行域变小,只会增加最小化目标值。影子价格非负。
- **= → 自由:** 同时是上界和下界;无法假设符号。

**一个统一的记法**:**影子价格符号 = "放松约束对目标的好处方向"**。
- 如果放松能让目标变好(更小,在 min 里)→ $p_i \ge 0$
- 如果放松反而让目标变差 → $p_i \le 0$
- 如果方向不确定(=) → 自由

记住这一句,你就不需要死记表格。

### 转换工作示例(最大化原问题)
$$\max 4x_1 + 5x_2 \;\text{s.t.}\; x_1 + 2x_2 \le 10,\; -x_1 + x_2 \ge 2,\; x_1 \ge 0,\; x_2 \text{ free}$$

应用最大化规则:
- 约束 1 (≤):对偶变量 $p_1 \le 0$
- 约束 2 (≥):对偶变量 $p_2 \ge 0$
- 变量 1 (≥0):对偶约束 $p_1 - p_2 \le 4$
- 变量 2 (自由):对偶约束 $2p_1 + p_2 = 5$

**逐步推导:**
- max 原问题 → 对偶是 min。
- 2 个约束 (RHS = 10, 2) → 2 个对偶变量 $p_1, p_2$,目标系数是 10 和 2。
- 2 个变量 ($x_1, x_2$,系数是 4 和 5)→ 2 个对偶约束,RHS 是 4 和 5。
- 约束矩阵 $A = \begin{pmatrix} 1 & 2 \\ -1 & 1 \end{pmatrix}$,转置后 $A^T = \begin{pmatrix} 1 & -1 \\ 2 & 1 \end{pmatrix}$——这就是对偶约束左边的系数矩阵。

**对偶问题:**
$$\min 10p_1 + 2p_2 \;\text{s.t.}\; p_1 - p_2 \le 4,\; 2p_1 + p_2 = 5,\; p_1 \le 0, p_2 \ge 0$$

注意 $p_1 \le 0$ 因为原问题约束 1 是 ≤(max 情况翻转后符号变负),$p_2 \ge 0$ 因为约束 2 是 ≥。变量 $x_2$ 自由 → 对偶约束 2 是等式 (=)。

## 6. 分步方法

### (a) 从应用题构造 LP
1. 识别决策变量(使用描述性名称)。
2. 写目标函数:max/min Σ c·x。
3. 列出约束(资源、市场、配比、非负性)。
4. 将每个约束表达为线性不等式/等式。
5. 验证单位。

**最常犯的错误**:跳过第 1 步,直接开始写。先把决策变量用文字精确定义(例如:"$x_B$ = 本周生产的 bun 数量,单位:个"),后面的所有公式都会自动对齐。

### (b) 机械地写出对偶问题
1. 识别原问题形式:min/max、约束类型、变量符号。
2. 为每个原问题约束创建一个对偶变量。
3. 翻转目标方向。
4. 转置:原问题约束 RHS → 对偶目标系数;原问题变量系数 → 对偶约束 RHS。
5. 应用转换表(根据原问题类型确定读取方向)。

### (c) 利用弱对偶验证最优性
1. 找到可行的原问题解 $x$,计算 $c^T x$。
2. 找到可行的对偶解 $p$,计算 $b^T p$。
3. 如果 $c^T x = b^T p$:两者都是最优。
4. 如果 $c^T x < b^T p$(最大化原问题):$x$ 可能不是最优。

**注意第 4 步的"可能"**——也可能是 $p$ 不是最优。光看不等式你只能排除"两者同时最优"的可能性,不能确定谁不对。

### (d) 使用互补松弛
1. 对每个原问题约束 $i$:如果 $p_i^* > 0$,则约束 $i$ 是紧的。
2. 对每个对偶约束 $j$:如果 $x_j^* > 0$,则约束 $j$ 是紧的。
3. 利用紧约束求解未知量。

**CS 在考试里的常见用法:** 题目给你最优 $x$,问你最优 $p$ 是什么。步骤:
1. 看哪些 $x_j > 0$ → 这些对应的对偶约束都是等式。
2. 看哪些原问题约束有 slack > 0 → 这些对应的 $p_i = 0$。
3. 把以上信息代进去,通常剩下的就是一个小型线性方程组,直接解。

### (e) "购买更多资源"的决策
1. 找到影子价格 $p_i^*$。
2. 与单位成本 $q$ 比较。
3. 如果 $q < p_i^*$ → 购买。每单位利润增益 = $p_i^* - q$。
4. 有效性:仅适用于小范围变化(灵敏度范围内)。

**做这道题的检查清单**:
- 单位换算了吗?(\$X/kg vs \$Y/g)
- 我比较的是同一种资源的市场价和影子价格,还是把不同资源弄混了?
- 题目说的"购买量"在灵敏度范围内吗?如果超出,影子价格不再有效。

## 7. 常见错误与陷阱

1. **忘记弱对偶要求双方都可行。** 如果你猜测的对偶解不可行,则无法应用弱对偶。
   - 为什么:整个证明链条 $c^T x \le p^T A x \le p^T b$ 依赖于 $A x \le b$ **和** $p \ge 0$ **和** $p^T A \ge c^T$。任何一个不成立,推不下去。

2. **忘记检查原问题方向。** Min 和 Max 在转换表中的符号规则是相反的。
   - 教训:在草稿纸的右上角先大写写下 "PRIMAL = MAX" 或 "PRIMAL = MIN",再开始用表。

3. **对非紧约束使用影子价格。** 松弛 > 0 意味着 $p_i = 0$。
   - 如果你算出某个紧约束的 $p_i = 0$,要么算错了,要么这个约束其实有 slack。

4. **在灵敏度范围之外应用影子价格。** 大幅变化会使线性近似失效;需要重新求解。
   - 安全经验:课堂上演示的"加 +10,000g 面粉"就破坏了线性预测。如果题目里 $\Delta b$ 看起来"很大",就要小心。

5. **忘记对偶变量的非负性。** 这继承自原问题约束类型。
   - 别瞎写 $p_i \ge 0$——根据原问题(min/max)和约束类型 (≤/≥/=) 决定符号。

6. **混淆 slack 与 surplus。** 对 ≤ 约束,slack = RHS − LHS。
   - 对 ≥ 约束叫 surplus = LHS − RHS,符号方向相反。

7. **把影子价格与市场价格混淆。** 影子价格是*隐含的*边际价值,而不是市场价格。
   - 影子价格是"系统内部"的估值,只在最优解附近有意义;市场价是外部世界的报价。两者的对比才能产生决策。

8. **对最大化原问题以错误方向读取转换表。** 始终先声明"原问题是 max"。

9. **忘记单位换算。** \$0.03/g ≠ \$30/kg... 请仔细检查。
   - \$30/kg = \$0.03/g(实际上相等!),\$5/kg = \$0.005/g。打开计算器,别口算。

10. **忽视强对偶验证。** 强对偶是你拥有的最简单的最优性检验。
    - 如果你算出原问题 = 对偶 + 两边都可行,可以直接收笔,不需要任何额外步骤。

## 8. 考试相关性

**模式 A — 对偶问题构造:** "写出以下 LP 的对偶。" 5 分钟。使用转换表。

**模式 B — 影子价格决策:** "X 的影子价格是 \$Y。要以 \$Z 购买更多吗?" 2 分钟。比较 Z 与 Y。注意单位!

**模式 C — 互补松弛推导:** "给定 $x^*$,哪些对偶约束是紧的?" 3 分钟。机械地应用。

**模式 D — 最优性验证:** "验证 $x$ 和 $p$ 都是最优的。" 3 分钟。检查可行性 + 目标值相等。

**值得学习的练习题:** LP Duality-I(玩具),LP Duality-II(布料)。

**时间分配提醒**:这四种模式都是"机械题"——按表/按规则推就行,不需要灵感。如果一道题超过 5 分钟你还没下笔,大概率是哪里读错题了——重读题目,而不是死磕公式。

## 9. 小练习

### Q1. 从最大化原问题构造对偶
$$\max 5x + 3y \;\text{s.t.}\; 2x + y \le 8,\; x + 2y \le 6,\; x, y \ge 0$$

**答案(对偶问题):**
$$\min 8p_1 + 6p_2 \;\text{s.t.}\; 2p_1 + p_2 \ge 5,\; p_1 + 2p_2 \ge 3,\; p_1, p_2 \ge 0$$

**推导步骤:** max → 对偶 min;两个约束 (8, 6) → 两个对偶变量,目标系数 8 和 6;两个变量 (5, 3) → 两个对偶约束,RHS 是 5 和 3;约束矩阵 $\begin{pmatrix}2&1\\1&2\end{pmatrix}$ 转置后还是它自己,所以对偶约束左边和原约束系数对称。

### Q2. 影子价格解读
给定最优 $p_1 = 2.2, p_2 = 0.6$。解读 $p_1$;放松约束 1 的影响?

**答案:** $p_1 = 2.2$ = 资源 1 的影子价格。将 RHS 从 8 增加到 9 使最优值增加约 \$2.2。

**一句话理解**:多 1 单位资源 1,目标增加 2.2;多 1 单位资源 2,目标增加 0.6。所以**资源 1 是更稀缺/更有价值的瓶颈**。如果只能多买一种,买资源 1。

### Q3. 互补松弛检验
原问题 $x = 4, y = 0$;对偶 $p_1 = 2.2, p_2 = 0.6$。用互补松弛 (CS) 验证。

**答案:** $x = 4 > 0$ → 对偶约束 1 紧:$2(2.2) + 0.6 = 5$ ✓。$y = 0$ → 对偶约束 2 可以有松弛($2.2 + 1.2 = 3.4 > 3$ ✓)。$p_1 > 0$ → 原问题约束 1 紧($2(4) + 0 = 8$ ✓)。$p_2 > 0$ → 原问题约束 2 应该紧($4 + 0 = 4 ≠ 6$,松弛=2)❌ 不一致。所以这些不是共同最优的。

**这道题展示的关键技巧**:CS 不只是"检验",也是"反证工具"。一旦你发现某条 CS 关系不成立(比如这里 $p_2 > 0$ 但约束 2 有 slack=2),你立刻知道**给的解不是最优**——不需要算目标值或重新求解。这在考试里很省时间。

### Q4. 是否购买更多资源
面粉影子价格 = \$0.03/g。报价 (a) \$0.02/g,(b) \$0.04/g,(c) \$0.03/g。

**(a)** 购买:净 +\$0.01/g。
**(b)** 不购买:净 −\$0.01/g。
**(c)** 无差异(边界)。

**思考边界情况 (c)**:严格来说在边界上,买和不买结果一样,但实务中你会偏向"不买"——因为购买可能有交易成本/物流成本,而且你不确定影子价格的精确度。

### Q5. 经济对偶推导
产品 A(\$10 利润,2 劳动力,1 机器),产品 B(\$15 利润,1 劳动力,3 机器),100 劳动力,80 机器。写出原问题和对偶问题。

**原问题:** $\max 10x_A + 15x_B$ s.t. $2x_A + x_B \le 100$,$x_A + 3x_B \le 80$,$x_A, x_B \ge 0$。

**对偶问题:** $\min 100p_L + 80p_M$ s.t. $2p_L + p_M \ge 10$,$p_L + 3p_M \ge 15$,$p_L, p_M \ge 0$。

解读:一个竞争对手以价格 $(p_L, p_M)$ 购买所有资源;他们需要每种产品的"购买价格"超过我们生产它所能获得的利润。

**展开经济叙事**:想象有个竞争对手叫 Yash,他想把你工厂里所有的劳动力和机器买走。他出价 $p_L$ 元/单位劳动力、$p_M$ 元/单位机器。
- 总出价:$100 p_L + 80 p_M$。
- 你愿意卖的条件:**对每种你能生产的产品**,Yash 出的钱要比你做这个产品赚的钱多——否则你宁可自己生产。
  - 做一个 A 用 2 劳动力 + 1 机器,赚 \$10 → Yash 必须出 $2 p_L + p_M \ge 10$。
  - 做一个 B 用 1 劳动力 + 3 机器,赚 \$15 → Yash 必须出 $p_L + 3 p_M \ge 15$。
- Yash 想花最少的钱说服你 → min $100 p_L + 80 p_M$。

强对偶告诉你:Yash 最少要花 = 你自己生产能赚的最多 = 同一个数。

## 10. 速查卡

- **弱对偶(永远成立):** max 原问题 ≤ min 对偶问题。
- **强对偶(在最优时):** 两者相等。
- **互补松弛:** $x_j > 0$ ↔ 对偶约束 $j$ 紧;$p_i > 0$ ↔ 原问题约束 $i$ 紧。
- **影子价格决策:** 当成本 < 影子价格时,购买资源。
- **转换表:** min 原问题从左到右读,max 原问题从右到左读。
- **Gurobi:** `constraint.pi` = 影子价格,`.X` = 变量值,`.slack` = RHS−LHS。

### 考试导向的口头提示

- **"强对偶是最简单的最优性检验。"** 如果你计算 原问题值 = 对偶值 且两者都可行 → 完成。比互补松弛更快。
- **"灵敏度范围很重要。"** 影子价格只在某个范围内成立。如果考试说"将面粉增加 50%",这超出了范围——你需要重新求解,而不是仅仅乘以影子价格。
- **面包店 LP 是按 LP(而非 IP)求解的**,因为对于数千个单位,将 113,478.248 → 113,478 最多损失 \$3。除非是是/否决策(产品组合、选址、TSP),否则默认使用整数松弛。
- **对偶是列生成 (column generation) 的前提**(不在考试范围内,但解释了为什么对偶是基础)。

### 通过重复强调的概念

- **通过约束的正线性组合得到上界的"配方"**(他讲了 3 遍,学生尝试给出了错误的上界,然后才纠正到对偶问题)。
- **互补松弛是双向的:**
  - 松弛 > 0 ⇒ 对偶变量 = 0
  - 对偶变量 > 0 ⇒ 约束是紧的
  - 许多学生忘记反方向。
- **影子价格 = 目标函数对约束 RHS 的导数。** 泰勒展开:$f(x+h) \approx f(x) + f'(x) h$。

### 起到澄清作用的口头解释

- **为什么我们不在 Gurobi 中添加非负性:** `addVar()` 默认下界为 0,上界为 ∞。你可以注释掉非负性这几行。
- **为什么 IP 会破坏对偶:** 有了整数约束,强对偶就失效了(你得到的是拉格朗日对偶——一个上下界,而非精确值)。PDF 笔记中提到过,但超出 W1 范围。不要尝试对整数规划应用影子价格。
- **经济对偶解读("Yash 卖原料"的故事):** 原问题 = "决定生产什么"。对偶问题 = "竞争对手购买所有原料;价格必须覆盖你通过烘焙每种产品所能赚的钱"。这解释了为什么对偶约束是 $\ge$(而非 $\le$),以及为什么 $p_i \ge 0$。

### 学生问答亮点

- **"我们必须显式求解对偶吗?"** 不需要——单纯形法/内点法在内部维护对偶。`m.optimize()` 同时给出两者。在约束引用上使用 `.pi`。
- **"黄油明明是好东西,为什么黄油的影子价格 = 0?"** 因为我们还有 1000g 未使用。松弛 > 0 ⇒ 价格 = 0。
- **"影子价格公式对任意 $\Delta b$ 都成立吗?"** 仅在灵敏度范围内。Gurobi 通过 `.SARHSLow` 和 `.SARHSHigh` 报告它。超出该范围时,其他约束会变成紧约束。

### 数值演算 — 从上界到对偶

构造上界的过程:
- 仅黄油约束:上界 = 26,000(过松 10 倍)
- 黄油 × 0.5:上界 = 13,000(过松 5 倍)
- 学生尝试乘子 0.1:给出无效上界 2,600(系数不满足 ≥)
- 学生纠正:面粉 × 0.1 + 糖 × 0.1:上界 = 8,500(有效,更紧)
- 通过求解对偶得到的最优上界:2,650(与原问题匹配!)

这就是苏格拉底式的动机:学生通过尝试权重发现对偶,然后对偶问题自然浮现。

**这个过程展示了对偶的"心理起源"**——不是从天而降的公式,而是"如何找最紧的上界"这个问题逼着你定义对偶变量。每一步学生猜的权重都对应一个对偶可行解,只是不一定最优;最优时,猜的权重就是 LP 解出的对偶解。

### 灵敏度验证(notebook 现场运行)

| ΔFlour (g) | 预测值(影子价格) | 实际值 | 是否匹配? |
|---|---|---|---|
| 0 | 2,650 | 2,650 | ✓ |
| +100 | 2,650 + 100(0.03) = 2,653 | 2,653 | ✓(精确) |
| +1,000 | 2,680 | 2,680 | ✓(精确,在边界) |
| +10,000 | 2,950 | < 2,950 | ❌(近似失效) |

**重点看最后一行**:+10,000g 时线性预测说 +\$300,实际增加不到 \$300——为什么?因为这时候**糖**变成了新的瓶颈(糖只有 5000g,做更多产品时早晚要用光糖)。一旦瓶颈切换,影子价格就变了。这就是"灵敏度范围"的真实含义。

### 口头标注的警告

1. **永远不要用负数乘以约束**——不等号会翻转,上界被破坏。始终 $p_i \ge 0$。
2. **当约束有松弛时不要应用影子价格。** 松弛 > 0 ⇒ 价格 = 0。
3. **大幅资源变化会使近似失效。** 使用影子价格时声明"假设小幅变化"。
4. **注意单位!** 影子价格 \$0.03/g vs 市场价格 \$5/kg = \$0.005/g。比较前先换算。
5. **负的对偶价格在标准形式中没有意义。**

### 教授风格中的考试策略

对于"这个 $(x^*, p^*)$ 是否最优?":
1. 检查 $x^*$ 在原问题中可行 且 $p^*$ 在对偶问题中可行。
2. 计算两个目标值。
3. 如果相等 → 两者都最优(强对偶)。停止。
4. 只有当你需要知道哪些约束是紧的时,才使用互补松弛。

**这套流程的关键是顺序**:**先做最简单的检查(强对偶),再做复杂的检查(CS)**。考试时间宝贵,而强对偶只要算两个目标值——10 秒搞定。CS 要逐条核对每个变量和约束的关系——可能 3 分钟。先用便宜的工具排除大部分情况。

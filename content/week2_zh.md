# 第二周 — 收益管理

## 0. 全景视角：航空业的故事

收益管理（Revenue management）源于航空业的一场危机。1978年，《航空公司放松管制法案》（Airline Deregulation Act）允许航空公司自由定价。像 PEOPLExpress 这样的低成本航司将主要航空公司的票价压低了 50-70%，抢走了休闲旅客市场。美国航空（American Airlines）副总裁 Robert Crandall 意识到：*一旦飞机已经起飞，多填一个座位的成本几乎为零。* 美国航空可以为休闲旅客提供"Ultimate Super Saver"超低折扣票，同时仍然向商务旅客收取高价。

**危机：** 1978 至 2002 年间，有 100 多家航空公司破产。**解决方案：** 容量控制（capacity control）——每个航班仅出售有限数量的折扣座位，并随着起飞临近动态决策。美国航空将其在 1988-1991 年间 14 亿美元的增量收入归功于 RM（收益管理）。这就是现代收益管理（yield management）的诞生。

## 1. 背景设定 — 为什么容量分配很难

航班：160 个座位，在 200 天内售出。
- 顾客 A（第 10 期）：想以 $80 购买经济舱座位
- 顾客 B（第 195 期）：愿意为商务舱支付 $500

**权衡（核心权衡）：**
- 现在接受 A：得到 $80，但失去卖给 B 的能力
- 等待 B：保留容量，但 B 可能永远不会出现

**机会成本（Opportunity cost）：** 一个座位的价值 = 如果现在使用它，你预期会失去的收入。

## 2. 单航段 RM 演示

### 设定
- 容量 $C = 160$ 座位
- $K = 10$ 个票价等级，收入 $r_j$（$100 到 $500）
- 时间范围 $T = 200$ 期
- 每期至多一个到达，等级 $j$ 出现的概率为 $p_j$
- 期望需求：$E[D_j] = T \cdot p_j$

### 短视策略（接受一切）
```
If b > 0: ACCEPT
Else: REJECT
```
为什么它失败：用 $80 的票价填满 160 个座位，导致后面没有空间卖 $500 的票价。平均收入约 $27,788。

### 确定性 LP
**决策：** $x_j$ = 接受的 $j$ 类请求数量。

**公式：**
$$\max \sum_{j=1}^{K} r_j x_j$$
$$\text{s.t. } \sum_{j=1}^{K} x_j \le C \quad \text{(capacity)}$$
$$x_j \le E[D_j] = T p_j \quad \text{(demand cap)}$$
$$x_j \ge 0$$

**数值示例：** $K=10$，收入 $[100,115,120,140,150,210,220,400,450,500]$，概率 $[0.16,0.16,0.14,0.1,0.1,0.08,0.06,0.05,0.04,0.02]$，$T=200$，$C=160$。

预测 = $T \cdot p$ = $[32,32,28,20,20,16,12,10,8,4]$。总计 = 182 > 160 容量。

**最优分配（Allocation）：** $[10, 32, 28, 20, 20, 16, 12, 10, 8, 4]$。LP 值 = $29,440。第 1 类（$100）被压缩到 10；更高等级获得完整预测。

## 3. 投标价格控制 投标价格控制

**投标价格** $\pi_\ell$ = 容量约束的影子价格（shadow price）= 多一个座位的边际价值。

### 决策规则
在时间 $t$，剩余 $b$ 个座位时：
1. 用剩余需求 $(T-t+1) p_j$ 求解 LP$(t, b)$
2. 得到容量约束的影子价格 $\pi_t(b)$
3. **若 $r_j \ge \pi_t(b)$ 且有座位可用，则接受请求 $j$**

### 直觉
- $\pi$ = 现在使用一个座位的机会成本（相对于保留给后续更高票价）
- 仅当票价超过机会成本时接受
- $\pi$ 随着 $t$ 接近起飞时间而增加（高票价到达的时间变少）
- $\pi$ 随着 $b$ 减少而增加（稀缺性）

### 数值示例
在 $t = 100$，$b = 50$ 座位，$\pi = 200$：
- 等级 A（$150）：$150 < 200$ → 拒绝
- 等级 B（$200）：$200 \ge 200$ → 接受
- 等级 C（$180）：$180 < 200$ → 拒绝

## 4. 单航段 Notebook — 单元格演示

**Cell 1：数据设置**
```python
revenue = [100,115,120,140,150,210,220,400,450,500]
probability = [0.16,0.16,0.14,0.1,0.1,0.08,0.06,0.05,0.04,0.02]
B = 160; T = 200
forecast = T * probability  # = [32,32,28,...,4]
```

**Cell 3：求解静态 LP**
```python
m = Model()
x = m.addVars(nFares, lb=0, ub=forecast)
seat_constr = m.addConstr(quicksum(x[i] for i in range(nFares)) <= B)
m.setObjective(quicksum(revenue[i]*x[i] for i in range(nFares)), GRB.MAXIMIZE)
m.optimize()
# LP_obj = 29440
```

**Cell 6：bpc() 函数**
```python
def bpc(b, t):
    seat_constr.rhs = b
    for i in range(nFares):
        x[i].ub = (T-t)*probability[i]
    m.optimize()
    return seat_constr.pi
```
- `bpc(160, 0)` → π = 100
- `bpc(20, 99)` → π = 210（稀缺性）

**Cell 13：模拟投标价格策略**
```python
for s in range(100):
    revenue_s, b = 0, 160
    for t in range(T):
        fare = arrival_sequence[t]
        if fare < nFares:  # not "no arrival"
            dual = bpc(b, t)
            if revenue[fare] >= dual:
                b -= 1
                revenue_s += revenue[fare]
# Mean revenue ≈ 29,329 (99.6% of LP bound)
```

**Cell 19：短视策略基准** → 平均收入 27,788。与投标价格的差距 = 5.5%。在固定成本 $26,000 的情况下，**利润提升 = 86%**。

## 5. 多航段 / 网络 RM

### 设定
多个航段共享容量：
- $M$ 个航段，各自容量 $b_\ell$
- $n$ 个行程（部分单航段，部分使用 2 个以上航段的多航段）
- 若 $a_{\ell i} = 1$，则行程 $i$ 使用航段 $\ell$

### LP
$$\max \sum_i r_i x_i$$
$$\text{s.t. } \sum_{i: a_{\ell i}=1} x_i \le b_\ell \quad \forall \ell$$
$$x_i \le E[D_i] = T p_i$$
$$x_i \ge 0$$

### 多航段的投标价格规则
行程 $i$ 使用集合 $S_i$ 中的航段。当且仅当满足以下条件时接受：
$$r_i \ge \sum_{\ell \in S_i} \pi_\ell$$

**示例：** 行程 BOS→SLC→LAX 使用航段 0 和 3。若 $\pi_0 = 50, \pi_3 = 40$，总投标价格 = 90。若票价 ≥ 90 则接受。

```python
itineraries_to_legs = [[0],[1],[2],[3],[0,2],[0,3],[1,2],[1,3]]
```

## 7. 期望到达数 — 数学

### 伯努利（离散时间）
- 每期，等级 $j$ 以概率 $p_j$ 到达
- 在 $[t, T]$ 内的期望到达数：$E[D_j(t,T)] = (T-t+1) p_j$

### 泊松（连续时间）
- 单位时间的速率 $\lambda_j$
- 在 $[t, T]$ 内的期望到达数：$E[N_{t,T,j}] = \lambda_j (T-t)$

## 8. 算法伪代码

```
b ← C
revenue ← 0
FOR t = 1 to T:
    Solve LP(t, b):
        max Σ r_i x_i
        s.t. Σ a_{ℓi} x_i ≤ b_ℓ  ∀ℓ
             x_i ≤ (T-t) p_i  ∀i
    π ← shadow prices of capacity constraints
    Observe request type i
    IF i is real AND r_i ≥ Σ_{ℓ in legs(i)} π_ℓ AND b[ℓ] > 0 for all ℓ:
        ACCEPT: revenue += r_i; b[ℓ] -= 1 for each ℓ in legs(i)
RETURN revenue
```

## 9. 常见错误 ⚠️

1. **混淆需求与分配。** 预测 = 最大需求；分配 = 最优接受数量。
2. **忽略机会成本。** 不要仅仅因为座位空着就接受——要与投标价格比较。
3. **静态 vs 动态混淆。** LP 给出静态分配；投标价格动态地重新求解。
4. **航段记账错误。** 多航段请求使用其经过航段的投标价格之**和**。
5. **将投标价格视为固定值。** $\pi_t(b)$ 随时间和容量变化。
6. **忘记需求上限。** 没有 $x_j \le E[D_j]$ 时，LP 会把所有容量都分配给最高票价。
7. **声称不可能的收入。** LP 值是可实现期望收入的**上界**（upper bound）。

## 11. 小练习

**Q1.** 酒店：100 间房，10 个晚上。等级 A（$80，p=0.05/晚），B（$120，0.06），C（$200，0.02）。写出 LP。

**答案：** $\max 80x_A + 120x_B + 200x_C$ s.t. $x_A + x_B + x_C \le 100$，$x_A \le 0.5, x_B \le 0.6, x_C \le 0.2$，$x \ge 0$。

**Q2.** $\pi = 180$。接受票价 $r_j = 200$？**是**（200 ≥ 180）。

**Q3.** 多航段票价 $85，使用 2 个航段，$\pi_1 = 50, \pi_2 = 40$。**拒绝**（$85 < 90$）。

**Q4.** LP 最优值 = $30k。策略声称收入 $35k。**不可能**——LP 是上界。

**Q5.** $T = 200, p_J = 0.05$。从 $t = 150$ 开始 J 类的期望到达数？$(200-150+1)(0.05) = 51 \times 0.05 = 2.55$。

## 12. 速查卡

| 概念 | 公式 |
|---|---|
| 容量 LP | $\max \sum r_j x_j$ s.t. $\sum x_j \le C$，$x_j \le E[D_j]$ |
| 投标价格 = | 容量约束的影子价格 |
| 单航段规则 | 当且仅当 $r_j \ge \pi_t(b)$ 时接受 |
| 多航段规则 | 当且仅当 $r_i \ge \sum_{\ell \in S_i} \pi_\ell$ 时接受行程 $i$ |
| 伯努利到达 | $E[D_j] = (T-t+1) p_j$ |
| 泊松到达 | $E[N] = \lambda_j (T-t)$ |
| LP 值 = | 期望收入的上界 |

**关键直觉：** 投标价格 = 机会成本。当且仅当票价 ≥ 机会成本时接受。每次请求时重新求解。

## 🎙️ 讲座录音洞见 — 幻灯片之外

来自 Misic 教授 4/8 的讲座——他讲了但幻灯片没有体现的内容。

### 考试导向的口头提示

- **"在考试中这是十分中的一分"** — 识别 max 还是 min（很简单但在压力下容易搞错）。
- **"期望到达数来自二项分布"** — $D_j$ ~ Binomial$(T, p_j)$。当 $T$ 较大时，根据中心极限定理近似为正态。但底层是二项分布。学生们把预测当成确定性的——这是错误的。
- **"对偶变量作为敏感性——这在今天非常重要。"** 投标价格的基础。
- **策略规范是难以处理的：** 对于 $T$ 期 ×（$B+1$）容量水平 ×（$n+1$）决策，可能的策略数为 $2^{T(B+1)(n+1)}$。LP 近似很重要，因为直接 DP 是不可行的。

### 通过重复强调的概念

**机会成本**（在整个讲座中重复 8 次以上）：
- "在这个时间点上，这个座位的价值是多少？"
- "如果你现在使用一个座位，你就失去了以后卖出它的机会。"
- "这是多一个座位的**边际**价值。"

这是把整个投标价格框架粘合在一起的**纽带**。

**短视策略作为基准：** 总是能售罄，但留下了未赚到的钱（$27,788 vs $29,329）。不要把"卖光所有容量"和"最大化收入"混为一谈。

**投标价格随时间和容量都变化** — 多个工作示例：
- `bpc(160, 0)` → $\pi = 100$（早期，满机 → 低机会成本）
- `bpc(160, 100)` → $\pi = 0$（晚期，满机 → 几乎没有时间等高票价）
- `bpc(20, 99)` → $\pi = 210$（晚期，几乎空机 → 高稀缺溢价）

许多学生把 $\pi$ 当作固定数字——这是错的。

### 真实世界的故事（提供背景）

**People Express 的破产：** 收入 10 亿美元但利润仅 6000 万美元（远低于预期的 3.5 亿美元）。为什么？只压低价格而没有容量控制——当 $500 的票价可用时，却用 $80 的票价填满飞机。这就是为什么 RM 很重要。

**旅行办公室的轶事：** 一位学生从上海到洛杉矶的航班被订成 SHG → HKG → TPE → LAX。COVID 时期的过境签证规则意味着该学生无法进入台湾。这说明了我们为什么要建模"10 个票价等级"——有 10+ 个隐藏的票价来源（公司办公室、Expedia、航空公司官网等），每个都有不同的规则。

### 口头澄清

**为什么分数分配在概念上是可以的：** "它不可实施，但有用。" 如果 $x_1 = 3.5$，意思是"在多次重复中，平均每个航班接受 3.5 个第 1 类请求"。解决了关于分数座位的困惑。

**Python 索引陷阱：** 幻灯片使用 $T - t + 1$（数学惯例，包含当前期的剩余期数）。Notebook 代码使用 `T - t`（零索引）。不要机械地复制。

**伯努利 vs 二项 vs 泊松的清晰区分：**
- 每期：等级 $j$ 以概率 $p_j$ 到达，各期独立 → 每期为伯努利。
- 在 $T$ 期内：等级 $j$ 的到达数 ~ Binomial$(T, p_j)$，均值 $Tp_j$。
- 连续时间的类比：速率为 $\lambda_j$ 的泊松过程，均值 $\lambda_j T$。

### 学生问答

- **"为什么不一直接受商务舱？"** 因为你**不知道**商务旅客是否会到达。如果你保留容量但只有经济舱到达，你就浪费了航班。**平均**地求解。
- **"航空公司实际如何优化？"** 模拟：20,000 次 LP 求解（100 次模拟 × 200 天）。真实航空公司：销售开放时一次，然后每晚更新。Notebook 使用 `m.params.OutputFlag = 0` 来抑制日志。
- **"投标价格依赖于时间吗？"** "非常依赖。也非常依赖于容量。" **两个**维度都重要。
- **"我们怎么知道接近最优？"** LP 值是上界。达到 29,329 vs 29,440 LP 是 0.37% 的差距。渐近最优——差距随问题规模缩小。
- **"超售（overbooking）怎么办？"** 修改座位约束：$0.8 x_i \le$ 座位数（假设 80% 出现率）。但可能会破坏线性。

### 关键利润洞察

收入提升 5.5%（29,329 vs 27,788）。但在 $26,000 的固定成本下：利润提升 **86%**（$3,329 vs $1,788）。

> *"收入上小百分比的提升带来利润上**大**百分比的提升"*，因为航空公司的固定成本巨大。这就是为什么 RM 在实践中重要。

### 口头标记的警告

1. **不要混淆预测与分配。** 预测 = $Tp$。分配 = 你选择接受的 $x_j$。
2. **LP 是上界，而非保证。** 如果一家公司声称他们达到了 LP 值，那是**不可信**的。
3. **投标价格不是常量** — 随 $t$ 和 $b$ 变化。
4. **对于多航段，将所用航段的投标价格求和。** 常见错误：将票价与单个航段的投标价格比较。
5. **影子价格可以为零。** 如果需求预测 < 容量，则 $\pi = 0$。
6. **在时间范围末期，你会变得绝望。** 当 $t \to T$ 而 $b > 0$ 时，$\pi \to 0$。

### 策略

1. 构建静态 LP（容量 + 需求上限）。
2. 求解，提取对偶变量（投标价格）。
3. 动态：在每次请求时，将票价与相关投标价格之和进行比较。
4. 当且仅当票价 ≥ 投标价格时接受。
5. 在每次请求时**重新求解** LP，使用更新后的剩余容量和时间。

**关键洞察：** 你**不是**直接求解最优策略。你求解 LP 并**使用**其对偶值来指导一个贪心的接受/拒绝规则。事实证明该规则是渐近最优的。

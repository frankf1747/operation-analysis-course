# 第二周 — 收益管理

## 0. 全景视角:航空业的故事

收益管理(Revenue management)源于航空业的一场危机。1978 年,《航空公司放松管制法案》(Airline Deregulation Act)允许航空公司自由定价。像 PEOPLExpress 这样的低成本航司将主要航空公司的票价压低了 50-70%,抢走了休闲旅客市场。美国航空(American Airlines)副总裁 Robert Crandall 意识到:*一旦飞机已经起飞,多填一个座位的成本几乎为零。* 美国航空可以为休闲旅客提供 "Ultimate Super Saver" 超低折扣票,同时仍然向商务旅客收取高价。

**危机:** 1978 至 2002 年间,有 100 多家航空公司破产。**解决方案:** 容量控制(capacity control)——每个航班仅出售有限数量的折扣座位,并随着起飞临近动态决策。美国航空将其在 1988-1991 年间 14 亿美元的增量收入归功于 RM(收益管理)。这就是现代收益管理(yield management)的诞生。

**为什么本周重要:** 这不是航空业独有的——酒店、邮轮、租车、广告位、电力市场、甚至 Uber 的动态定价,本质上都在做同样的事:**用一个固定的、易消逝的容量,服务一批价值参差不齐的需求**。掌握 RM 的思维方式,你就能解决一大类商业问题。

## 1. 背景设定 — 为什么容量分配很难

航班:160 个座位,在 200 天内售出。
- 顾客 A(第 10 期):想以 \$80 购买经济舱座位
- 顾客 B(第 195 期):愿意为商务舱支付 \$500

**权衡(核心权衡):**
- 现在接受 A:得到 \$80,但失去卖给 B 的能力
- 等待 B:保留容量,但 B 可能永远不会出现

**机会成本(Opportunity cost):** 一个座位的价值 = 如果现在使用它,你预期会失去的收入。

**这就是 RM 的核心思维方式**:不要问"这个座位值 \$80 吗?"——要问"这个座位**保留给未来**值多少钱?"。如果未来期望收入(机会成本)是 \$120,而现在的报价只有 \$80,你应该拒绝。这听起来反直觉——*拒绝一个有钱顾客*——但只要机会成本算对了,长期收入会更高。

注意机会成本不是固定的——它随时间和剩余座位**动态变化**:
- 临近起飞 → 高票价旅客来的概率降低 → 机会成本下降 → 更愿意接受低票价
- 座位即将售罄 → 稀缺性 → 机会成本上升 → 只接受高票价

后面的投标价格 $\pi_t(b)$ 就是把这种动态机会成本**精确量化**。

## 2. 单航段 RM 演示

### 设定
- 容量 $C = 160$ 座位
- $K = 10$ 个票价等级,收入 $r_j$ (\$100 到 \$500)
- 时间范围 $T = 200$ 期
- 每期至多一个到达,等级 $j$ 出现的概率为 $p_j$
- 期望需求:$E[D_j] = T \cdot p_j$

**为什么把时间离散成 200 期?** 因为大部分预订系统是按"事件触发"的——每次有人查询/下单就是一期。把整个 200 天分成 200 期,每期最多来一个顾客,数学上最干净。如果改成连续时间,公式换成泊松过程(后面会讲)。

### 短视策略(接受一切)
```
If b > 0: ACCEPT
Else: REJECT
```
为什么它失败:用 \$80 的票价填满 160 个座位,导致后面没有空间卖 \$500 的票价。平均收入约 \$27,788。

**直观失败模式**:短视策略只看"现在能不能赚钱",不看"卖给现在的人会损失未来什么"。当低票价旅客先到(早期常见),你会把座位"用便宜了"——飞机起飞时确实满了,但收入远低于潜力。

**关键 lesson**:**"飞机满了"不等于"收入最大"**。后面会用数字证明,短视收入 = \$27,788,LP 上界 = \$29,440,差距 5.5%。

### 确定性 LP
**决策:** $x_j$ = 接受的 $j$ 类请求数量。

**公式:**
$$\max \sum_{j=1}^{K} r_j x_j$$
$$\text{s.t. } \sum_{j=1}^{K} x_j \le C \quad \text{(capacity)}$$
$$x_j \le E[D_j] = T p_j \quad \text{(demand cap)}$$
$$x_j \ge 0$$

**读懂这个 LP:**
- **目标**:总收入 = $\sum r_j x_j$,最大化。
- **第一个约束(capacity)**:接受的总人数不超过座位数。这是物理约束。
- **第二个约束(demand cap)**:对每个票价等级,接受数不能超过期望到达数。如果一个等级期望只来 10 个人,LP 不会假装能接受 100 个。
- **非负性**:不能"接受负数个人"。

**为什么需要 demand cap?** 没有它,LP 会把全部 160 个座位都分配给最高票价等级 \$500——但实际上 \$500 的顾客一共才来 4 个 ($T \cdot p_{10} = 200 \cdot 0.02 = 4$)。这显然不现实。

**数值示例:** $K=10$,收入 $[100,115,120,140,150,210,220,400,450,500]$,概率 $[0.16,0.16,0.14,0.1,0.1,0.08,0.06,0.05,0.04,0.02]$,$T=200$,$C=160$。

预测 = $T \cdot p$ = $[32,32,28,20,20,16,12,10,8,4]$。总计 = 182 > 160 容量。

**最优分配(Allocation):** $[10, 32, 28, 20, 20, 16, 12, 10, 8, 4]$。LP 值 = \$29,440。第 1 类 (\$100) 被压缩到 10;更高等级获得完整预测。

**怎么读这个分配:** 总需求 182 > 容量 160,所以必须"砍掉"22 个请求。LP 的最优策略是:**砍最低票价的**(第 1 类 \$100),把 32 个压缩到 10。其他等级全部满足。这就是 LP 自动发现的最优 trade-off——"放弃便宜的,保留贵的"。

## 3. 投标价格控制

**投标价格** $\pi_\ell$ = 容量约束的影子价格(shadow price)= 多一个座位的边际价值。

**回顾第一周**:影子价格 = 放松约束一个单位,目标值增加多少。这里把容量约束的影子价格起了个名字叫 "投标价格",但本质上**就是第一周学的影子价格**。

### 决策规则
在时间 $t$,剩余 $b$ 个座位时:
1. 用剩余需求 $(T-t+1) p_j$ 求解 LP$(t, b)$
2. 得到容量约束的影子价格 $\pi_t(b)$
3. **若 $r_j \ge \pi_t(b)$ 且有座位可用,则接受请求 $j$**

**这条规则在做什么:**
- $\pi_t(b)$ 是"现在用掉这个座位的隐含成本"——如果我把它留给未来,期望能多赚 $\pi_t(b)$。
- 当前票价 $r_j$ 是"现在卖掉的明确收益"。
- 净收益 = $r_j - \pi_t(b)$。
- 净收益 ≥ 0 → 接受;< 0 → 拒绝。

这就是经济学里最干净的边际决策框架——**只在边际收益 ≥ 边际成本时行动**。

### 直觉
- $\pi$ = 现在使用一个座位的机会成本(相对于保留给后续更高票价)
- 仅当票价超过机会成本时接受
- $\pi$ 随着 $t$ 接近起飞时间而增加(高票价到达的时间变少)
- $\pi$ 随着 $b$ 减少而增加(稀缺性)

**等等,上面说"$\pi$ 随 $t$ 增加",但也说"随 $b$ 减少而增加"——这两个方向矛盾吗?** 不矛盾——它们是**两个独立的维度**:
- **固定 $b$,$t$ 增加(临近起飞)**:理论上 $\pi$ 应该**下降**(高票价旅客来不及到了)。
- **固定 $t$,$b$ 减少(座位变少)**:$\pi$ **上升**(稀缺溢价)。
- 实际中两者交互——临近起飞且只剩 20 个座位,$\pi$ 反而可能比早期满机时还高。

后面 Notebook 里 `bpc(160, 0) = 100`、`bpc(20, 99) = 210` 这两个数就在演示这两个方向。

### 数值示例
在 $t = 100$,$b = 50$ 座位,$\pi = 200$:
- 等级 A (\$150):$150 < 200$ → 拒绝
- 等级 B (\$200):$200 \ge 200$ → 接受
- 等级 C (\$180):$180 < 200$ → 拒绝

**这个例子展示的关键点**:**不是所有票价高于某个阈值的都接受**——而是和**当时此刻的** $\pi$ 比较。$t = 50$ 时同样三个请求决策可能完全不同(因为 $\pi$ 不同)。这就是"动态"的含义。

## 4. 单航段 Notebook — 单元格演示

**Cell 1:数据设置**
```python
revenue = [100,115,120,140,150,210,220,400,450,500]
probability = [0.16,0.16,0.14,0.1,0.1,0.08,0.06,0.05,0.04,0.02]
B = 160; T = 200
forecast = T * probability  # = [32,32,28,...,4]
```

注意 `forecast = T * probability`——如果 `probability` 是 numpy array 才能这样写;list 不行。课堂代码假设 numpy。

**Cell 3:求解静态 LP**
```python
m = Model()
x = m.addVars(nFares, lb=0, ub=forecast)
seat_constr = m.addConstr(quicksum(x[i] for i in range(nFares)) <= B)
m.setObjective(quicksum(revenue[i]*x[i] for i in range(nFares)), GRB.MAXIMIZE)
m.optimize()
# LP_obj = 29440
```

**关键技巧**:`addVars(nFares, lb=0, ub=forecast)` 一次创建 10 个变量,**把需求上限作为变量上界写进去了**——这样就不需要单独写 10 条 demand cap 约束。Gurobi 把变量上界当作"隐式约束"处理,效率更高。

**Cell 6:bpc() 函数**
```python
def bpc(b, t):
    seat_constr.rhs = b
    for i in range(nFares):
        x[i].ub = (T-t)*probability[i]
    m.optimize()
    return seat_constr.pi
```
- `bpc(160, 0)` → π = 100
- `bpc(20, 99)` → π = 210(稀缺性)

**bpc 在做什么:** "bpc" = bid-price computation。给定当前剩余容量 $b$ 和剩余时间 $T - t$,**重新求解 LP** 并返回容量约束的影子价格。
- `seat_constr.rhs = b` → 更新容量上限。
- `x[i].ub = (T-t)*probability[i]` → 更新每个等级的剩余期望需求。
- `m.optimize()` → 重新求解。
- `seat_constr.pi` → 提取容量约束的对偶值,就是投标价格 $\pi$。

**两个数字对比:**
- `bpc(160, 0) = 100`:开始时(满机、200 期都没用),$\pi$ 只有 100——相对低,因为时间充足、座位也够。
- `bpc(20, 99) = 210`:半路上 (用了 99 期、只剩 20 个座位),$\pi$ 飙到 210——稀缺。

记住 $\pi$ 是个**函数** $\pi(b, t)$,不是常数。

**Cell 13:模拟投标价格策略**
```python
for s in range(100):
    revenue_s, b = 0, 160
    for t in range(T):
        fare = arrival_sequence[t]
        if fare < nFares:  # not "no arrival"
            dual = bpc(b, t)
            if revenue[fare] >= dual:
                b -= 1
                revenue_s += revenue[fare]
# Mean revenue ≈ 29,329 (99.6% of LP bound)
```

**这段在做什么:** 100 次蒙特卡洛模拟。每次模拟里 200 期,**每一期都重新算一次 $\pi$**,然后用决策规则决定是否接受。最终平均收入 \$29,329——非常接近 LP 上界 \$29,440(只差 0.4%)。

**为什么 LP 是上界?** 因为 LP 假设你**事前知道精确的期望需求**,并能"分数地"接受请求。现实中需求是随机的,接受是 0/1 的,所以实际收入 ≤ LP 值。

**Cell 19:短视策略基准** → 平均收入 27,788。与投标价格的差距 = 5.5%。在固定成本 \$26,000 的情况下,**利润提升 = 86%**。

**这个数字最值得记**:5.5% 的收入提升 → 86% 的利润提升。原因:航空公司固定成本巨大(燃油、人工、飞机折旧),收入的小百分比提升几乎全部转化为利润。**这就是为什么 RM 在实务中如此重要**——它不是"边际优化",而是"决定盈亏"。

## 5. 多航段 / 网络 RM

### 设定
多个航段共享容量:
- $M$ 个航段,各自容量 $b_\ell$
- $n$ 个行程(部分单航段,部分使用 2 个以上航段的多航段)
- 若 $a_{\ell i} = 1$,则行程 $i$ 使用航段 $\ell$

**真实世界的痛点:** 现实中很少有"BOS 直飞 LAX"那种单航段——大部分长途行程是"BOS → SLC → LAX"两段。卖一张 BOS-LAX 的票同时**占用两段的座位**,所以两段的容量是耦合的。这就是为什么需要网络模型。

### LP
$$\max \sum_i r_i x_i$$
$$\text{s.t. } \sum_{i: a_{\ell i}=1} x_i \le b_\ell \quad \forall \ell$$
$$x_i \le E[D_i] = T p_i$$
$$x_i \ge 0$$

**和单航段的区别:** 容量约束从 1 条变成 $M$ 条(每段一条)。每个行程 $x_i$ 出现在它经过的**所有**航段约束里。其他都一样。

### 多航段的投标价格规则
行程 $i$ 使用集合 $S_i$ 中的航段。当且仅当满足以下条件时接受:
$$r_i \ge \sum_{\ell \in S_i} \pi_\ell$$

**示例:** 行程 BOS→SLC→LAX 使用航段 0 和 3。若 $\pi_0 = 50, \pi_3 = 40$,总投标价格 = 90。若票价 ≥ 90 则接受。

**为什么是加总?** 因为多航段行程**同时消耗两段的容量**——你失去的机会成本是两段的和。这是网络 RM 最容易出错的地方:**不能拿票价和单段的 $\pi$ 比**——一定要和**所用航段的 $\pi$ 之和**比。

**陷阱举例**:如果票价 = \$70,$\pi_0 = 50, \pi_3 = 40$:
- ❌ 错误推理:"\$70 > \$50,接受"(只看一段)。
- ✅ 正确推理:"\$70 < \$90 (50+40),拒绝"(总机会成本)。

## 7. 期望到达数 — 数学

### 伯努利(离散时间)
- 每期,等级 $j$ 以概率 $p_j$ 到达
- 在 $[t, T]$ 内的期望到达数:$E[D_j(t,T)] = (T-t+1) p_j$

**记法说明**:$T - t + 1$ 是因为**包含当前期 $t$**。比如 $t=1, T=200$,剩余期数是 200(不是 199)。Notebook 里有时用 `T - t`(零索引),小心区分。

### 泊松(连续时间)
- 单位时间的速率 $\lambda_j$
- 在 $[t, T]$ 内的期望到达数:$E[N_{t,T,j}] = \lambda_j (T-t)$

### 伯努利 vs 二项 vs 泊松 — 快速对照

| 模型 | 适用 | 每期/单位时间到达数 | 总到达数 |
|---|---|---|---|
| 伯努利 (Bernoulli) | 每"期"最多一个到达 | $D_t \in \{0, 1\}$,概率 $p$ | 二项 $\text{Bin}(T, p)$,均值 $Tp$ |
| 泊松 (Poisson) | 连续时间,可同时多个 | 速率 $\lambda$ | 泊松 $\text{Poisson}(\lambda T)$,均值 $\lambda T$ |

**实务中的选择**:离散事件用伯努利/二项;连续时间(如打车请求、网页点击)用泊松。LP 公式里用的就是均值($Tp$ 或 $\lambda T$),所以两个模型给出同样的 LP。

## 8. 算法伪代码

```
b ← C
revenue ← 0
FOR t = 1 to T:
    Solve LP(t, b):
        max Σ r_i x_i
        s.t. Σ a_{ℓi} x_i ≤ b_ℓ  ∀ℓ
             x_i ≤ (T-t) p_i  ∀i
    π ← shadow prices of capacity constraints
    Observe request type i
    IF i is real AND r_i ≥ Σ_{ℓ in legs(i)} π_ℓ AND b[ℓ] > 0 for all ℓ:
        ACCEPT: revenue += r_i; b[ℓ] -= 1 for each ℓ in legs(i)
RETURN revenue
```

**读懂这个伪代码:**
1. 每一期开始:用当前剩余容量 $b$ 和剩余时间求一次 LP,提取 $\pi$。
2. 观察这一期实际来的请求是什么类型。
3. 如果票价 $r_i$ 超过这次行程经过所有航段的 $\pi$ 之和,且每段都还有座位 → 接受。
4. 接受后,经过的**每个**航段都减一个座位(`b[ℓ] -= 1`)。

**复杂度警告**:每期都求一次 LP,总共 $T$ 次。如果 $T = 200, M = 10, n = 50$,需要 200 次中等规模 LP。可行但慢。实务中常用"每天求一次"代替"每事件求一次"。

## 9. 常见错误 ⚠️

1. **混淆需求与分配。** 预测 = 最大需求;分配 = 最优接受数量。
   - 为什么会错:预测 32 个 \$100 顾客,LP 只接受 10 个——学生看到"32"以为是 LP 解。

2. **忽略机会成本。** 不要仅仅因为座位空着就接受——要与投标价格比较。
   - 这是短视策略的本质错误。

3. **静态 vs 动态混淆。** LP 给出静态分配;投标价格动态地重新求解。
   - LP 解 $x$ 不是策略——它只是分配建议。真正的策略是"每期用最新 $\pi$ 做决策"。

4. **航段记账错误。** 多航段请求使用其经过航段的投标价格之**和**。
   - 见上一节"陷阱举例"。考试最常错的地方之一。

5. **将投标价格视为固定值。** $\pi_t(b)$ 随时间和容量变化。
   - 看到 `bpc(160, 0) = 100`、`bpc(20, 99) = 210` 就明白了——不是同一个数。

6. **忘记需求上限。** 没有 $x_j \le E[D_j]$ 时,LP 会把所有容量都分配给最高票价。
   - 后果:LP 算出全部 160 个座位卖 \$500 → 80,000(不现实)。

7. **声称不可能的收入。** LP 值是可实现期望收入的**上界**(upper bound)。
   - 如果别人告诉你他们达到了或超过 LP 值,要么数据有错,要么需求被低估了。

## 10. 考试相关性

模拟考试幻灯片明确列出了 5 种 RM 问题类型:

2. "设计一个使用 LP 进行动态接受/拒绝的算法。"
3. "在时间 t,剩余 X 个座位时,影子价格 = Y。哪些请求类型应接受?"

5. "某人声称策略可实现 \$X。LP 值为 \$Y。可信吗?"

**应对策略**:
- 类型 1:把"卖什么"做决策变量,"用什么资源"做约束。容量是 ≤,需求上限是 ≤,非负性别忘。
- 类型 2:写伪代码——LP + 提取 $\pi$ + 决策规则 + 更新状态。
- 类型 3:直接比较 $r_j$ vs $\pi$,逐个判断。注意单位。
- 类型 4:就是个乘法:$(T - t + 1) p_j$。注意"剩余期数"是否包含当前期。
- 类型 5:LP 是上界——超过 LP 不可能,接近 LP 可信。

## 11. 小练习

**Q1.** 酒店:100 间房,10 个晚上。等级 A (\$80,p=0.05/晚),B (\$120,0.06),C (\$200,0.02)。写出 LP。

**答案:** $\max 80x_A + 120x_B + 200x_C$ s.t. $x_A + x_B + x_C \le 100$,$x_A \le 0.5, x_B \le 0.6, x_C \le 0.2$,$x \ge 0$。

**注意陷阱**:每个等级的期望需求是 $T \cdot p = 10 \cdot p$,所以 A 的上限是 $10 \cdot 0.05 = 0.5$——不是 0.05。如果你写 0.05,等于说"10 个晚上只准接受 0.05 个 A 顾客",显然错了。

**Q2.** $\pi = 180$。接受票价 $r_j = 200$?**是**(200 ≥ 180)。

边际收益 = 200 − 180 = 20 > 0,所以接受。

**Q3.** 多航段票价 \$85,使用 2 个航段,$\pi_1 = 50, \pi_2 = 40$。**拒绝**($85 < 90$)。

注意要**加总**:$50 + 40 = 90$,然后比较 $85 < 90$。如果你只和 $\pi_1 = 50$ 比,你会错误地接受。

**Q4.** LP 最优值 = \$30k。策略声称收入 \$35k。**不可能**——LP 是上界。

如果真的拿到 \$35k,要么 LP 模型错了(漏了需求/容量),要么测算数据有问题。

**Q5.** $T = 200, p_J = 0.05$。从 $t = 150$ 开始 J 类的期望到达数?$(200-150+1)(0.05) = 51 \times 0.05 = 2.55$。

"从 $t = 150$ 开始"包含 $t=150, 151, ..., 200$,共 51 期(注意是 +1)。

## 12. 速查卡

| 概念 | 公式 |
|---|---|
| 容量 LP | $\max \sum r_j x_j$ s.t. $\sum x_j \le C$,$x_j \le E[D_j]$ |
| 投标价格 = | 容量约束的影子价格 |
| 单航段规则 | 当且仅当 $r_j \ge \pi_t(b)$ 时接受 |
| 多航段规则 | 当且仅当 $r_i \ge \sum_{\ell \in S_i} \pi_\ell$ 时接受行程 $i$ |
| 伯努利到达 | $E[D_j] = (T-t+1) p_j$ |
| 泊松到达 | $E[N] = \lambda_j (T-t)$ |
| LP 值 = | 期望收入的上界 |

**关键直觉:** 投标价格 = 机会成本。当且仅当票价 ≥ 机会成本时接受。每次请求时重新求解。

## 🎙️ 讲座录音洞见 — 幻灯片之外

来自 Misic 教授 4/8 的讲座——他讲了但幻灯片没有体现的内容。

### 考试导向的口头提示

- **"在考试中这是十分中的一分"** — 识别 max 还是 min(很简单但在压力下容易搞错)。
- **"期望到达数来自二项分布"** — $D_j$ ~ Binomial$(T, p_j)$。当 $T$ 较大时,根据中心极限定理近似为正态。但底层是二项分布。学生们把预测当成确定性的——这是错误的。
- **"对偶变量作为敏感性——这在今天非常重要。"** 投标价格的基础。
- **策略规范是难以处理的:** 对于 $T$ 期 ×($B+1$)容量水平 ×($n+1$)决策,可能的策略数为 $2^{T(B+1)(n+1)}$。LP 近似很重要,因为直接 DP 是不可行的。

### 通过重复强调的概念

**机会成本**(在整个讲座中重复 8 次以上):
- "在这个时间点上,这个座位的价值是多少?"
- "如果你现在使用一个座位,你就失去了以后卖出它的机会。"
- "这是多一个座位的**边际**价值。"

这是把整个投标价格框架粘合在一起的**纽带**。如果考试只让你记一个词,就是"机会成本"。

**短视策略作为基准:** 总是能售罄,但留下了未赚到的钱(\$27,788 vs \$29,329)。不要把"卖光所有容量"和"最大化收入"混为一谈。

**投标价格随时间和容量都变化** — 多个工作示例:
- `bpc(160, 0)` → $\pi = 100$(早期,满机 → 低机会成本)
- `bpc(160, 100)` → $\pi = 0$(晚期,满机 → 几乎没有时间等高票价)
- `bpc(20, 99)` → $\pi = 210$(晚期,几乎空机 → 高稀缺溢价)

许多学生把 $\pi$ 当作固定数字——这是错的。

**怎么记这三个数:**
- 早 + 满 → $\pi$ 中等(还有时间,但座位也紧)。
- 晚 + 满 → $\pi$ 低(没时间等高票价了,反正满了)。
- 晚 + 空 → $\pi$ 高(座位稀缺,而且时间也不多,但每个剩余座位都很珍贵)。

### 真实世界的故事(提供背景)

**People Express 的破产:** 收入 10 亿美元但利润仅 6000 万美元(远低于预期的 3.5 亿美元)。为什么?只压低价格而没有容量控制——当 \$500 的票价可用时,却用 \$80 的票价填满飞机。这就是为什么 RM 很重要。

**旅行办公室的轶事:** 一位学生从上海到洛杉矶的航班被订成 SHG → HKG → TPE → LAX。COVID 时期的过境签证规则意味着该学生无法进入台湾。这说明了我们为什么要建模"10 个票价等级"——有 10+ 个隐藏的票价来源(公司办公室、Expedia、航空公司官网等),每个都有不同的规则。

### 口头澄清

**为什么分数分配在概念上是可以的:** "它不可实施,但有用。" 如果 $x_1 = 3.5$,意思是"在多次重复中,平均每个航班接受 3.5 个第 1 类请求"。解决了关于分数座位的困惑。

**Python 索引陷阱:** 幻灯片使用 $T - t + 1$(数学惯例,包含当前期的剩余期数)。Notebook 代码使用 `T - t`(零索引)。不要机械地复制。

**伯努利 vs 二项 vs 泊松的清晰区分:**
- 每期:等级 $j$ 以概率 $p_j$ 到达,各期独立 → 每期为伯努利。
- 在 $T$ 期内:等级 $j$ 的到达数 ~ Binomial$(T, p_j)$,均值 $Tp_j$。
- 连续时间的类比:速率为 $\lambda_j$ 的泊松过程,均值 $\lambda_j T$。

### 学生问答

- **"为什么不一直接受商务舱?"** 因为你**不知道**商务旅客是否会到达。如果你保留容量但只有经济舱到达,你就浪费了航班。**平均**地求解。
- **"航空公司实际如何优化?"** 模拟:20,000 次 LP 求解(100 次模拟 × 200 天)。真实航空公司:销售开放时一次,然后每晚更新。Notebook 使用 `m.params.OutputFlag = 0` 来抑制日志。
- **"投标价格依赖于时间吗?"** "非常依赖。也非常依赖于容量。" **两个**维度都重要。
- **"我们怎么知道接近最优?"** LP 值是上界。达到 29,329 vs 29,440 LP 是 0.37% 的差距。渐近最优——差距随问题规模缩小。
- **"超售(overbooking)怎么办?"** 修改座位约束:$0.8 x_i \le$ 座位数(假设 80% 出现率)。但可能会破坏线性。

### 关键利润洞察

收入提升 5.5%(29,329 vs 27,788)。但在 \$26,000 的固定成本下:利润提升 **86%**(\$3,329 vs \$1,788)。

> *"收入上小百分比的提升带来利润上**大**百分比的提升"*,因为航空公司的固定成本巨大。这就是为什么 RM 在实践中重要。

**这条洞察可以记一辈子**——任何"高固定成本 + 低边际成本"的行业都适用:酒店、邮轮、电影院、SaaS 软件、广告位。RM 不是航空业的把戏,是这类商业模型的通用工具。

### 口头标记的警告

1. **不要混淆预测与分配。** 预测 = $Tp$。分配 = 你选择接受的 $x_j$。
2. **LP 是上界,而非保证。** 如果一家公司声称他们达到了 LP 值,那是**不可信**的。
3. **投标价格不是常量** — 随 $t$ 和 $b$ 变化。
4. **对于多航段,将所用航段的投标价格求和。** 常见错误:将票价与单个航段的投标价格比较。
5. **影子价格可以为零。** 如果需求预测 < 容量,则 $\pi = 0$。
6. **在时间范围末期,你会变得绝望。** 当 $t \to T$ 而 $b > 0$ 时,$\pi \to 0$。

### 策略

1. 构建静态 LP(容量 + 需求上限)。
2. 求解,提取对偶变量(投标价格)。
3. 动态:在每次请求时,将票价与相关投标价格之和进行比较。
4. 当且仅当票价 ≥ 投标价格时接受。
5. 在每次请求时**重新求解** LP,使用更新后的剩余容量和时间。

**关键洞察:** 你**不是**直接求解最优策略。你求解 LP 并**使用**其对偶值来指导一个贪心的接受/拒绝规则。事实证明该规则是**渐近最优**的——也就是说当问题规模 $T, C$ 趋大时,这条规则得到的收入和真正最优策略的差距占比 → 0。这是 RM 理论上最深刻的结果之一。

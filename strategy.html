Question Patterns (Direct from Prof's Slides)

LP Duality

  • Formulate this problem as a linear program.
  • Formulate the dual problem of this LP.
  • Given a primal solution and a dual solution, is the primal optimal? (Use weak duality.)
  • Given the optimal dual, what does this imply about resource A's usage? (Complementary slackness.)
  • Suppose we increase quantity of this resource by 100 units. How much does profit change?

Revenue Management

  • Formulate the capacity control problem as an LP.
  • Design an algorithm using the LP for dynamic accept/reject.
  • At time $t$ with $X$ units of resource 1, $Y$ units of resource 2, which request types accept?
  • Expected number of arrivals of customer type 5 over horizon?
  • Someone claims algorithm achieves expected revenue 110, LP value is 100. Believable?

Newsvendor

  • Sorted data of historical demands, $p = 10$, $c = 3$. Recommended order?
  • Given density function, $p$, $c$ — what order quantity? (Integrate to CDF, invert.)
  • Historical data with regression tree. Given new context, how much to order?
  • Linear regression output. Recommend order quantity. (Conditional normal.)

Location

  • Formulate this problem as max coverage / p-median / p-center.
  • Write two different formulations of p-median. Which is better and why?
  • Modify to require each location serves at most $K$ customers.

Assortment

  • Given utility table, calculate choice probabilities for $S = \{1, 3, 4\}$.
  • Calculate expected per-customer revenue of $S = \{1, 3, 4\}$.
  • Upper bound on best possible revenue.
  • Formulate the general assortment problem as an IP.
  • Code + output (LP relaxation). Claim of revenue $X$. Believable?

Pricing

  • Linear demand model parameters. Calculate demand for given prices. Calculate revenue.
  • Formulate pricing as IP.
  • Partial formulation given. Which constraints are missing?

Network Flows

  • Word description of supply chain. Formulate as LP. Numbers explicit, no Σ.
  • Decide a route A to Z. With side constraints. Formulate as IP.

TSP

  • TSP formulation missing SECs — why does it fail?
  • Given solution $x$ and Python code, what is output of getSubtours?
  • From getSubtours output, write SECs added by lazy callback.
  • Word description of problem. Formulate as IP (recognize TSP).

What to Memorize

  • Primal→Dual conversion table (constraints↔variables)
  • Newsvendor critical fractile formulas
  • Triangular distribution PDF, CDF, and inverse (likely provided — but be safe)
  • Flow conservation equation: inflow + supply = outflow + demand
  • TSP degree=1 + SEC ≤ |S| − 1
  • Bid-price decision rule: fare ≥ Σ shadow prices
  • Side-constraint patterns (if A then B; at most K; etc. — see Templates page)

What to Understand

  • Why degree constraints alone fail in TSP (subtours allowed)
  • Why log-transform converts max-reliability to shortest path
  • Why customized bound > achievable revenue (per-segment customization)
  • Why shadow price = marginal value at boundary
  • Why LP relaxation gives upper bound for IP
  • Why strong constraints give faster solves (tighter LP)

What to Practice

  • Write LP formulations completely (define every variable; expand all constraints; no Σ if asked).
  • Trace Python code by hand — especially getSubtours and regression tree leaf prediction.
  • Integrate piecewise-defined densities to compute expected profit (triangular newsvendor).
  • Identify problem type from a single sentence description.
  • Do all 9 practice problems WITHOUT looking at solutions first.

Time Management on Exam Day

  1. First 5 minutes: Skim the whole exam. Identify easy problems.
  2. Do easy first. Bank points before tackling long problems.
  3. For formulation problems: Always start by defining variables clearly. Half the points often come from that.
  4. For "no Σ" questions: Take your time enumerating. Skipping a variable loses big points.
  5. For trace problems: Draw the cycles visually. Don't just stare at code.
  6. Show work: Even if final answer is wrong, partial credit for correct setup.
  7. Reserve 15 minutes at end to review every answer.

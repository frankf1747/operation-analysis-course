Must-Know Formulas (drill these tonight)

  1. $F(Q^*) = c_u / (c_u + c_o) = (p-c)/p$
  2. Bid-price decision: $r_j \ge \pi$? Accept : Reject
  3. Flow conservation: $\sum_j x_{ji} + b_i = \sum_j x_{ij}$ at every node
  4. Triangular CDF: $F(x) = (x-a)^2 / [(b-a)(c-a)]$ for $x \le b$; $1 - (c-x)^2/[(c-b)(c-a)]$ for $x > b$
  5. Empirical quantile: ⌈$n\alpha$⌉-th sorted value
  6. TSP SEC: $\sum_{i \in S} \sum_{j \in S, j \ne i} x_{ij} \le |S| - 1$
  7. Choice probability: sum of $\lambda_k$ over segments where $i$ is first choice
  8. Customized UB: $\sum_k \lambda_k \max_{i: u_{k,i} > u_{k,0}} r_i$
  9. Pricing demand: $d_i = \alpha_i - \beta_i p_i + \sum_{j \ne i} \gamma_{ij} p_j$

Must-Know Decision Procedures

QuestionProcedure
"Is primal optimal?"(a) primal feasible? (b) dual feasible? (c) same objective? If all yes → optimal. If dual infeasible → NO conclusion.
"Should we buy more X?"Compare per-unit cost vs shadow price. Buy iff cost < shadow price.
"How much will revenue change if resource changes by $\Delta$?"$\Delta R \approx p^* \cdot \Delta b$.
"Upper bound on assortment revenue?"(a) Customized: $\sum \lambda_k \max r_i$ over relevant products. OR (b) LP relaxation.
"Why does TSP need SECs?"Degree constraints alone allow disconnected subtours (small cycles).
"Is offering all products optimal?"Usually NO. Cannibalization. Need IP.
"Accept this RM request?"Compute (or use given) bid price $\pi$. Accept iff fare ≥ $\pi$ AND capacity available.
"Compute newsvendor Q*?"(1) compute $\alpha$, (2) identify distribution, (3) invert $F$.

Quick Sanity Checks

  • Newsvendor $\alpha < 0.5$ → $Q^* < $ median (order less). $\alpha > 0.5$ → $Q^* > $ median.
  • Shadow price = 0 → constraint has slack (resource not fully used).
  • Shadow price > 0 → constraint is tight (resource fully used).
  • Max-reliability: logs of $r < 1$ are NEGATIVE. Max = LEAST negative sum.
  • TSP SECs must include BOTH $x_{ij}$ AND $x_{ji}$.
  • Pricing: $\beta > 0$ always; own-price contributes $-\beta p^2$ (subtracted).
  • Choice prob: ALWAYS include no-purchase option in argmax.
  • Network: SUM of $b_i$ must equal 0 (balanced).

Triangular Distribution Quick Reference

Tr$[a, b, c]$: a=low, b=mode, c=high.

Inverse CDF:

  • $F(b) = (b-a)/(c-a)$ — check which branch.
  • If $\alpha \le F(b)$: $Q^* = a + \sqrt{\alpha(b-a)(c-a)}$
  • If $\alpha > F(b)$: $Q^* = c - \sqrt{(1-\alpha)(c-b)(c-a)}$

Mean: $(a+b+c)/3$.
